## 1. 模块定位

资源总览是生产监控层的聚合视角，帮助 VPP 运营人员快速掌握全系统或指定区域/电站的发电、负荷、可调空间和调度执行情况。它与 M02-02 电站监控构成上下钻取关系：资源总览看全局，电站监控看单站。

**VPP 运营核心链路**

```
光伏出力（实时+预测）──→ 负荷情况 ──→ 可调空间（负荷 - 出力）──→ 策略执行
```

本模块只关注这条链路上的指标。

## 4. 共享筛选栏

三个 Tab 共享同一个筛选条件，切换 Tab 时保留筛选状态。

```json
{
  "region": "华南区域",
  "stationId": "PV-001",
  "date": "2026-03-27",
  "granularity": "15m"
}
```

- 区域筛选过滤电站列表
- 选择单个电站时相当于电站监控的简要版

## 5. 三个 Tab 视图

### 5.1 出力概况

回答的问题：全系统/指定电站的发电和可调能力是否健康？

| # | 区域 | 内容 |
|---|---|---|
| 1 | KPI 卡片组（6 个） | 实时总功率(MW)、当日发电量(MWh)、可调总容量(MW)、在线率(%)、告警数、电站数 |
| 2 | 聚合功率曲线 | 所有电站聚合的实际功率曲线 + 可调区间带（ECharts 面积+折线） |
| 3 | 电站运行摘要表 | 各电站：名称、实时功率、日发电量、可调容量、在线率、状态。**行点击 → 切到电站监控 Tab 并定位** |

### 5.2 负荷与出力

回答的问题：各电站的负荷和出力关系如何？可调空间有多少？

| # | 区域 | 内容 |
|---|---|---|
| 1 | KPI 卡片组（4 个） | 总负荷(MW)、总出力(MW)、总可调空间(MW)、可调利用率(%) |
| 2 | 负荷 vs 出力对比曲线 | 双 Y 轴：负荷曲线 + 光伏出力曲线，面积填充展示可调空间 |
| 3 | 可调空间变化趋势 | 可调空间 = 负荷 - 出力 的时序曲线 |
| 4 | 电站可调空间排名 | 各电站可调空间从大到小排序，辅助调度决策 |

**聚合逻辑**

| 数据 | 聚合方式 |
|---|---|
| 总负荷 | 所有电站负荷求和 |
| 总出力 | 所有电站光伏出力求和 |
| 总可调空间 | 总负荷 - 总出力 |
| 可调利用率 | 已调用量 / 可调总量 |

### 5.3 调度执行

回答的问题：策略是否执行到位？收益如何？

| # | 区域 | 内容 |
|---|---|---|
| 1 | KPI 卡片组（4 个） | 策略执行率(%)、今日响应次数、累计响应电量(kWh)、今日收益(元) |
| 2 | 执行趋势曲线 | 目标功率 vs 实际功率 + 可调容量上限线 |
| 3 | 风险提示区 | 偏差超限电站、即将到期策略 |
| 4 | 调度执行记录表 | 时间、电站、策略、目标、实际、偏差、收益 |

## 7. 数据流

```
页面初始化:
  created() → loadOverviewData(region, stationId, date)

Tab 切换:
  @tab-change(tabKey) → loadTabData(tabKey, filters)

筛选变更:
  @filter-change(filters) → loadTabData(currentTab, filters)

摘要表行点击:
  @row-click(station) → 切换二级 Tab 到"电站监控"，传递 stationId
```

## 8. 后端接口（API）

| # | 接口 | 参数 | 说明 |
|---|---|---|---|
| 1 | GET /api/pvms/production-monitor/meta | - | 初始化：区域列表、电站列表、默认选中 |
| 2 | GET /api/pvms/production-monitor/overview | region, stationId | 出力概况 KPI + 聚合曲线 |
| 3 | GET /api/pvms/production-monitor/load-output | region, stationId, date | 负荷 vs 出力数据 |
| 4 | GET /api/pvms/production-monitor/dispatch | region, stationId, date | 调度执行数据 |

当前前端使用 mock 数据，接口格式已按上述规范定义，后端就绪后切换即可。

## 9. 样式规范

- 深色主题，使用 `--pvms-*` 系列 CSS 变量
- KPI 卡片使用 `AppMetricCard` 公共组件
- 图表配色对齐 `DashboardPowerCurve`（深色背景、蓝绿主色）
- 表格选中行使用 teal 半透明背景 `rgba(6, 162, 153, 0.15)`，不使用白色背景
- 图表最小高度 520px
