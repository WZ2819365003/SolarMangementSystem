## 1. 技术架构

```
┌─────────────────────────────────────────────────┐
│                   宿主应用 (VPP 主平台)            │
│       qiankun 微前端容器，提供 _tsx.apirequest     │
└──────────────────────┬──────────────────────────┘
                       │ mount(props)
┌──────────────────────▼──────────────────────────┐
│              PVMS 前端子应用                       │
│  ┌──────────┐ ┌──────────┐ ┌────────────────┐   │
│  │ Vue Router│ │  Vuex    │ │ Host Bridge    │   │
│  │ (SPA路由) │ │ (状态管理)│ │ (请求适配层)   │   │
│  └────┬─────┘ └────┬─────┘ └───────┬────────┘   │
│       │            │               │             │
│  ┌────▼────────────▼───────────────▼──────────┐  │
│  │          业务模块 (Pages + Components)       │  │
│  │  dashboard │ production-monitor │ stations  │  │
│  │  forecast  │ strategy │ devices │ alarms    │  │
│  └────────────────────────────────────────────┘  │
│       │                                          │
│  ┌────▼────────────────────────────────────────┐ │
│  │  UI 层：Element UI + ECharts + 高德地图       │ │
│  └─────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

### 关键设计决策

| 决策 | 原因 |
|---|---|
| Vue 2 而非 Vue 3 | 宿主平台基于 Vue 2，微前端子应用需版本对齐 |
| qiankun 协议兼容 | 导出 bootstrap/mount/unmount 生命周期，被宿主按需挂载/卸载 |
| 三模式请求适配 | 同一套代码支持宿主注入请求、独立后端请求和纯前端 mock 三种模式 |
| 按业务模块组织目录 | 每个功能模块自包含 pages/components/views，降低模块间耦合 |
| ECharts 5 全量引入 | 需要 liquidfill（水球图）等非标组件，按需引入收益不明显 |

## 3. 应用入口与生命周期

**入口文件**：`src/main.js`

### 3.1 独立运行模式

当 `window.__POWERED_BY_QIANKUN__` 为 `false` 时，直接调用 `render()` 创建 Vue 实例，挂载到 `#app`。

### 3.2 微前端挂载模式

导出三个 qiankun 生命周期函数：

```javascript
export async function bootstrap() {}  // 首次加载
export async function mount(props) {   // 挂载
  render(props)  // props 含 container、baseUrl、tenantName 等
}
export async function unmount() {      // 卸载
  vueInst.$destroy()
  vueInst.$el.innerHTML = ''
}
```

`mount` 时从 `props` 中获取宿主注入的配置，包括：
- `container`：宿主提供的 DOM 容器
- `baseUrl`：路由基础路径（微前端模式下非 `/`）
- `tenantName`：租户名称

### 3.3 Vuex Store

`src/store/index.js` 导出工厂函数 `createStore()`，在 `render()` 中调用。

Store 状态：

| State | 来源 | 用途 |
|---|---|---|
| `appName` | settings.js | 应用名称，用于侧边栏和文档标题 |
| `menus` | settings.js | 左侧菜单列表 |
| `runtime` | bridge.js + mount props | 运行时上下文（宿主模式、租户名等） |

## 4. 路由设计

**路由文件**：`src/router/index.js`

导出工厂函数 `createRouter(base)`，支持传入基础路径。

### 4.1 路由表结构

所有业务路由均为 `ShellLayout` 的子路由，共享侧边栏和顶栏壳：

| 路径 | 组件 | viewKey | 说明 |
|---|---|---|---|
| `/dashboard/overview` | OverviewPage | — | 综合监控中心首页 |
| `/production-monitor/overview` | ProductionMonitorPage | `overview` | 出力概况 |
| `/production-monitor/output` | ProductionMonitorPage | `output` | 出力分析 |
| `/production-monitor/load` | ProductionMonitorPage | `load` | 负荷与出力 |
| `/production-monitor/dispatch` | ProductionMonitorPage | `dispatch` | 调度执行 |
| `/production-monitor/station` | ProductionMonitorPage | `station` | 电站监控 |
| `/forecast/overview` | ForecastPage | `overview` | 预测总览 |
| `/forecast/adjustable` | ForecastPage | `adjustable` | 可调能力分析 |
| `/forecast/accuracy` | ForecastPage | `accuracy` | 精度评估 |
| `/strategy/list` | StrategyPage | `list` | 策略列表 |
| `/strategy/config` | StrategyPage | `config` | 策略配置 |
| `/strategy/revenue` | StrategyPage | `revenue` | 收益分析 |
| `/devices/monitor` | DeviceMonitorPage | — | 设备监控 |
| `/alarms/center` | AlarmCenterPage | — | 告警中心 |

### 4.2 viewKey 机制

生产监控、预测分析和策略管理三个模块使用同一个 Page 组件，通过 `$route.meta.viewKey` 决定内部渲染哪个子视图组件。这种设计让切换 Tab 时不会销毁模块级状态（meta、query 等），同时保持 URL 语义清晰。

### 4.3 路由元信息

每条路由的 `meta` 包含：
- `title`：面包屑和标题栏文字
- `section`：所属业务板块
- `summary`：页面功能一句话说明
- `activeMenu`：左侧菜单高亮路径

## 5. 数据请求层

### 5.1 三模式请求分流

**核心文件**：`src/shared/host/bridge.js`

```
请求发起 (api/pvms.js 调用 request())
         │
         ├─ 宿主注入了 _tsx.apirequest?  ──→ 走宿主请求通道
         │
         ├─ VUE_APP_REQUEST_MODE=mock?   ──→ 走前端 mock handlers
         │
         └─ 否则                          ──→ 走 axios 请求后端 API
```

**后端请求模式**（默认）：axios baseURL 为 `/api`，由 `vue.config.js` 的 devServer proxy 将 `/api` 转发到 `http://127.0.0.1:8091`。

**Mock 模式**：设置环境变量 `VUE_APP_REQUEST_MODE=mock` 后，所有请求走 `src/shared/mock/` 下的 JavaScript 函数，返回纯前端构造的模拟数据。

### 8.2 数据流

- 页面 `created()` 时调用 `fetchProductionMonitorMeta()` 获取资源单元列表
- 根据 URL query 的 `resourceUnitId` 或默认值选中资源单元
- 根据当前 `viewKey` 调用对应的数据接口
- 切换 Tab 时路由变化 → watcher 触发 `loadCurrentView()`

### 8.3 viewKey → 数据接口映射

| viewKey | 接口函数 |
|---|---|
| `overview` | `fetchProductionMonitorOverview` |
| `output` | `fetchProductionMonitorOutput` |
| `load` | `fetchProductionMonitorLoad` |
| `dispatch` | `fetchProductionMonitorDispatch` |

cd frontend

devServer 配置（`vue.config.js`）：

```bash
npm run build
```

输出到 `frontend/dist/`，`publicPath` 为 `/pvms/`。

前端构建产物通过 nginx 提供服务：
- 基础镜像：`nginx:1.27-alpine`
- 静态文件路径：`/usr/share/nginx/html/pvms`
- 容器端口：`80`，映射到宿主 `6618`
