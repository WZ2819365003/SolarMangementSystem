## 1. 数据库总览

| 域 | 表前缀 | 表数量 | 说明 |
|---|---|---|---|
| 驾驶舱 | `dashboard_` | 4 | 电站地理信息、实时状态快照、告警快照、VPP 节点 |
| 生产监控 | `pm_` | 6 | 资源单元、站点、实时快照、气象、15 分钟曲线、调度记录 |
| 电站档案 | `sa_` | 7 | 公司、站点详情、历史曲线、逆变器、逆变器告警、站点策略 |
| 预测分析 | `fc_` | 6 | 预测模型、站点绑定、预测序列、误差样本、月度精度、可调窗口 |
| 策略管理 | `sg_` | 6 | 策略主表、分时段、执行日志、电价、日收益、模拟快照 |

**字符集**：所有表使用 `utf8mb4`，引擎为 `InnoDB`。

**初始化脚本**：
- 建表：`backend/src/main/resources/schema.sql`（兼容 MySQL 和 H2）
- 生产种子数据：`backend/src/main/resources/mysql-data.sql`
- 测试种子数据：`backend/src/main/resources/data.sql`（H2 语法）

## 2. 驾驶舱域（dashboard_）

### 2.1 dashboard_station — 电站基础信息

存储所有电站的地理和容量信息，是驾驶舱模块的核心主表。

| 字段 | 类型 | 主键 | 说明 |
|---|---|---|---|
| id | VARCHAR(32) | PK | 电站编号，如 `PV-001` |
| name | VARCHAR(128) | | 电站名称 |
| resource_unit_id | VARCHAR(32) | | 所属资源单元编号 |
| resource_unit_name | VARCHAR(128) | | 所属资源单元名称 |
| region | VARCHAR(64) | | 所属区域，如「华南园区」 |
| longitude | DOUBLE | | 经度（WGS84） |
| latitude | DOUBLE | | 纬度（WGS84） |
| address | VARCHAR(255) | | 详细地址 |
| capacity_kwp | DECIMAL(12,2) | | 装机容量（kWp） |

### 2.2 dashboard_station_status_snapshot — 电站状态快照

存储每个电站的最新运行状态和 KPI 指标。与 `dashboard_station` 一对一关联。

| 字段 | 类型 | 主键 | 说明 |
|---|---|---|---|
| station_id | VARCHAR(32) | PK, FK→dashboard_station | 电站编号 |
| status | VARCHAR(32) | | 运行状态：normal / warning / fault / maintenance / offline |
| realtime_power_kw | DECIMAL(12,2) | | 实时功率（kW） |
| today_energy_kwh | DECIMAL(12,2) | | 今日累计发电量（kWh） |
| today_revenue_cny | DECIMAL(12,2) | | 今日收益（元） |
| availability | DECIMAL(5,2) | | 可用率（%） |
| health_grade | VARCHAR(16) | | 健康等级：A / B / C / D |
| snapshot_time | TIMESTAMP | | 快照时间 |

### 2.3 dashboard_alarm_snapshot — 告警快照

存储驾驶舱展示的实时告警事件列表。

| 字段 | 类型 | 主键 | 说明 |
|---|---|---|---|
| id | VARCHAR(64) | PK | 告警事件编号 |
| station_id | VARCHAR(32) | FK→dashboard_station | 关联电站 |
| event_time | TIMESTAMP | | 告警发生时间 |

| alarm_type | VARCHAR(64) | | 告警类型，如「通讯中断」「温度超限」 |
| description | VARCHAR(255) | | 告警描述 |
| status | VARCHAR(32) | | 处理状态：待处理 / 处理中 / 已关闭 |
| owner | VARCHAR(64) | | 责任人 |
| suggestion | VARCHAR(255) | | 处理建议 |

### 2.4 dashboard_vpp_node_snapshot — VPP 节点快照

存储虚拟电厂节点的可调容量状态。

| 字段 | 类型 | 主键 | 说明 |

### 3.5 pm_station_curve_15m — 15 分钟出力曲线

存储站点每天 96 个时段（15 分钟粒度）的出力、负荷和预测数据。

| 字段 | 类型 | 主键 | 说明 |
|---|---|---|---|
| station_id | VARCHAR(32) | PK(1/3), FK→pm_station | 站点编号 |
| biz_date | DATE | PK(2/3) | 业务日期 |
| time_slot | INT | PK(3/3) | 时段序号（0-95，对应 00:00-23:45） |
| load_kw | DECIMAL(12,2) | | 负荷（kW） |
| pv_power_kw | DECIMAL(12,2) | | 光伏实际出力（kW） |
| forecast_power_kw | DECIMAL(12,2) | | 预测出力（kW） |
| baseline_power_kw | DECIMAL(12,2) | | 基线出力（kW） |
| irradiance | INT | | 辐照度（W/m²） |
| temperature | DECIMAL(5,2) | | 温度（°C） |

**时段计算**：time_slot = 小时 × 4 + 分钟 ÷ 15。例如 09:30 → slot 38。
