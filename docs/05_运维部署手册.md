## 1. 环境要求

本项目包含两套前端运行环境和一套后端环境，请勿混淆。

| 用途 | 软件 | 版本要求 | 说明 |
|---|---|---|---|
| 前端开发/构建 | Node.js | **14.16.0** | 严格锁定，见 `frontend/package.json` engines |
| 前端开发/构建 | npm | **6.14.11** | 随 Node 14.16.0 附带 |
| Playwright E2E | Node.js | **18+** | 见 `frontend/tests/playwright/package.json` |
| 后端运行 | Java | **21** | 见 `backend/pom.xml` |
| 后端构建/测试 | Maven | **3.x** | 需可执行 `mvn` |
| 数据库 | MySQL | **8.x** | localhost:3306，库名 `pvms` |

### 1.1 Node 版本管理建议

由于前端主应用和 Playwright 的 Node 版本不同，建议使用 `nvm`（Unix）或 `nvm-windows`（Windows）来切换 Node 版本。或者像本项目的启动脚本一样，直接使用绝对路径指定 Node/npm 可执行文件。

## 3. 数据库准备

### 3.1 创建数据库

```bash
mysql -u root -p123456 -e "CREATE DATABASE IF NOT EXISTS pvms CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

### 3.2 建表

```bash
mysql -u root -p123456 --default-character-set=utf8mb4 pvms < backend/src/main/resources/schema.sql
```

`schema.sql` 中使用 `DROP TABLE IF EXISTS` + `CREATE TABLE`，可反复执行（会清除旧数据）。

### 3.3 灌入种子数据

```bash
mysql -u root -p123456 --default-character-set=utf8mb4 pvms < backend/src/main/resources/mysql-data.sql
```

**注意**：必须加 `--default-character-set=utf8mb4`，否则中文字段会因 GBK 编码报错。

### 3.4 验证数据

```bash
mysql -u root -p123456 -e "SELECT COUNT(*) FROM pvms.dashboard_station;"
# 预期结果：8（8 个电站）

mysql -u root -p123456 -e "SELECT COUNT(*) FROM pvms.sa_inverter;"
# 预期结果：16（16 台逆变器）
```

### 3.5 连接信息

| 参数 | 值 |
|---|---|
| URL | `jdbc:mysql://localhost:3306/pvms` |
| 用户名 | `root` |
| 密码 | `123456` |
| 字符集 | `utf8mb4` |
| 时区 | `Asia/Shanghai` |

配置文件位置：`backend/src/main/resources/application.yml`

## 5. 前端启动

### 5.1 安装依赖

```bash
cd frontend
npm install --scripts-prepend-node-path=true
```

**关于 hs-dtk-ui**：这是一个团队内部 UI 组件库，通过本地 tgz 路径安装：

```json
"hs-dtk-ui": "file:../../vpp-main/node_modules/hs-dtk-ui/hs-dtk-ui-0.1.29.tgz"
```

如果本机没有 `vpp-main` 目录，`npm install` 会失败。解决方案：
1. 找到 tgz 包，放到正确的相对路径
2. 或将 tgz 复制到项目内，修改 `package.json` 中的路径
3. 或联系团队获取包源配置

### 5.2 启动开发服务器

```bash
npm run serve
```

服务运行在 `http://127.0.0.1:6618`。devServer 会自动将 `/api` 请求代理到后端 `http://127.0.0.1:8091`。

### 5.3 建议启动顺序

1. 先启动后端，确认 `/api/system/health` 正常
2. 再启动前端
3. 浏览器访问 `http://127.0.0.1:6618/dashboard/overview`

## 6. 高德地图密钥配置

综合监控中心（M01）的 GIS 地图使用高德地图 Web JS API。如果不配置密钥，地图区域会显示空白或报错，但**不影响其他模块的正常使用**。

### 6.1 申请高德密钥

1. 访问 [高德开放平台](https://lbs.amap.com/)
2. 注册/登录 → 进入控制台 → 应用管理 → 创建应用
3. 为应用添加 Key：
   - 服务平台选择 **Web 端(JS API)**
   - 记录 Key 和安全密钥

### 6.2 配置环境变量

在 `frontend/` 目录下创建 `.env` 文件（基于 `.env.example`）：

```env
VUE_APP_AMAP_KEY=你的高德Key
VUE_APP_AMAP_SECURITY_CODE=你的安全密钥
VUE_APP_AMAP_VERSION=2.0
```

### 6.3 安全码说明

2021 年 12 月之后申请的高德 Key **必须**配合安全码使用。安全码通过 `window._AMapSecurityConfig` 注入（代码中已处理）。

### 6.4 不配置地图时

如果暂时不需要地图功能，可以不配置。此时：
- 综合监控中心的地图区域显示空白
- 控制台可能出现高德 API 加载错误
- 其他所有模块正常运行

## 8. Docker 部署

### 8.1 预构建部署包

项目提供两套开箱即用的 Docker 部署包，位于 `docker/` 目录下：

| 目录 | 架构 | 镜像标签 | 适用场景 |
|---|---|---|---|
| `docker/docker_out/` | x86_64 (amd64) | `pvms-backend:2.0.0`、`pvms-frontend:2.0.0` | 标准 x86 服务器 |
| `docker/docker_out_arm/` | ARM64 (aarch64) | `pvms-backend:2.0.0-arm64`、`pvms-frontend:2.0.0-arm64` | AWS Graviton、鲲鹏、Apple Silicon 等 |

每个部署包均为自包含目录，包含：
- `images/` — 三个镜像 tar（后端、前端、MySQL 8.0）
- `sql/` — 建表 DDL + 种子数据
- `init.sh` — MySQL 首次启动初始化脚本
- `docker-compose.yml` — 编排文件
- `README.md` — 部署手册

### 8.2 快速部署（推荐）

将对应架构的部署包上传到服务器后：

```bash
cd /opt/pvms

# 1. 加载镜像
docker load -i images/mysql-8.0.tar
docker load -i images/pvms-backend.tar
docker load -i images/pvms-frontend.tar

# 2. (可选) 修改数据库密码
echo "MYSQL_ROOT_PASSWORD=你的强密码" > .env

# 3. 启动
docker compose up -d
```

详细部署步骤请参见各部署包内的 `README.md`。

### 8.3 镜像信息

| 镜像 | 基础镜像 | 端口 | 说明 |
|---|---|---|---|
| pvms-frontend | nginx:1.27-alpine | 80 | 静态文件在 `/usr/share/nginx/html/pvms`，内置 nginx 反向代理配置 |
| pvms-backend | eclipse-temurin:21-jre-alpine | 8091 | Spring Boot Fat JAR |
| mysql:8.0 | 官方 MySQL | 3306 | 通过 init.sh 自动导入 schema 和种子数据 |

### 8.4 访问

- 前端：`http://<服务器IP>/pvms/`（根路径自动跳转）
- 后端健康检查：`http://<服务器IP>:8091/api/system/health`

### 8.5 容器管理

```bash
docker compose ps              # 查看状态
docker compose logs -f pvms-backend  # 查看后端日志
docker compose restart pvms-backend  # 重启后端
docker compose down            # 停止（保留数据）
docker compose down -v         # 停止并删除数据卷
```

### 8.6 注意事项

- 三个容器通过 Docker 内部网络通信，外部只需开放 **80** 端口
- MySQL 密码通过环境变量 `MYSQL_ROOT_PASSWORD` 配置，默认值 `pvms@Prod2026`，**生产环境务必修改**
- MySQL 数据持久化在 Docker 命名卷 `pvms-mysql-data` 中，容器重启不丢失

## 9. 生产构建与发布

### 9.1 后端构建

```bash
cd backend
mvn clean package -DskipTests
# 产物：target/pvms-backend-2.0.0.jar
```

运行：

```bash
java -jar target/pvms-backend-2.0.0.jar
```

### 9.2 前端构建

```bash
cd frontend
npm run build
# 产物：dist/ 目录
```

构建产物的 `publicPath` 为 `/pvms/`，部署到 nginx 时需配置对应的 `location /pvms/`。

### 9.3 nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location /pvms/ {
        alias /path/to/frontend/dist/;
        try_files $uri $uri/ /pvms/index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8091;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 12. 常见问题排查

### 12.1 npm install 失败

**症状**：`npm install` 报错找不到 `hs-dtk-ui` 的 tgz 文件。

**原因**：`package.json` 中 `hs-dtk-ui` 依赖引用了本地路径 `file:../../vpp-main/node_modules/hs-dtk-ui/hs-dtk-ui-0.1.29.tgz`。

**解决**：
1. 确认本机存在 `vpp-main` 目录且包含该 tgz 文件
2. 或从团队获取 tgz 包，复制到正确位置
3. 或修改 `package.json` 中的路径指向实际位置

### 12.2 Node 版本不对

**症状**：前端启动报语法错误或依赖安装失败。

**原因**：前端主应用严格要求 Node 14.16.0。

**解决**：使用 nvm 切换到 Node 14.16.0，或使用绝对路径：
```bash
C:\Users\zhuow\tools\node-v14.16.0-win-x64\npm.cmd run serve
```

### 12.3 地图页面报错

**症状**：综合监控中心地图区域空白或控制台报高德 API 错误。

**原因**：未配置高德地图密钥。

**解决**：参见[第 6 节 高德地图密钥配置](#6-高德地图密钥配置)。

### 12.4 后端连接 MySQL 失败

**症状**：启动后端报 `Communications link failure` 或 `Unknown database 'pvms'`。

**排查步骤**：
1. 确认 MySQL 服务正在运行：`mysql -u root -p123456 -e "SELECT 1"`
2. 确认 `pvms` 数据库已创建：`mysql -u root -p123456 -e "SHOW DATABASES"`
3. 确认已执行 schema.sql 建表
4. 确认 `application.yml` 中的连接信息正确

### 12.5 后端能起但返回空数据

**症状**：接口返回成功但 `data` 中的列表为空。

**原因**：未灌入种子数据。

**解决**：
```bash
mysql -u root -p123456 --default-character-set=utf8mb4 pvms < backend/src/main/resources/mysql-data.sql
```

### 12.6 中文数据乱码或插入失败

**症状**：`Incorrect string value: '\xE6\xB7\xB1...'`

**原因**：MySQL 客户端字符集不是 utf8mb4。

**解决**：导入数据时加 `--default-character-set=utf8mb4` 参数。

### 12.7 后端测试失败

**症状**：`mvn test` 报错 `Unknown database 'pvms'`。

**原因**：测试类缺少 `@ActiveProfiles("test")`，导致使用了 MySQL 配置而非 H2。

**解决**：确保所有测试类都有 `@ActiveProfiles("test")` 注解。

### 12.8 Playwright 起不来

**排查**：
1. 确认使用了 Node 18+（不是 Node 14）
2. 确认执行过 `npm run install:browsers`
3. 确认前端开发服务器端口 6618 可用
