# 系统总览与环境

## 1. 系统定位

项目名称：光伏管理系统  
形态：前端微应用 + Spring Boot 后端  
用途：面向虚拟电厂 / 聚合商场景，统一查看电站地图、生产监控、预测分析、策略管理、设备监测和告警中心。

## 2. 当前阶段判断

当前更准确的状态是“开发联调版 / 内嵌 H2 演示版”。

已经具备的东西：

- 前端主页面和模块路由已经齐全。
- 后端已经不只是纯 mock 接口，生产监控、电站监控、预测分析、策略管理、设备监测、告警中心都接入了 H2 内嵌库。
- 前后端在开发环境下默认走真实后端代理，不再默认走前端 mock。

还没有完全做到的东西：

- 外部真实业务系统接入还没有做，现在大部分基础数据仍然是 H2 种子数据。
- 权限、审计日志、生产部署、真实数据库、实时消息链路都还没有完善。
- 部分旧接口仍然保留 mock 服务。


- Password：空

## 6. 开发环境请求链路

前端统一通过 `frontend/src/shared/host/bridge.js` 发请求。

当前逻辑：

- 如果在宿主环境下并且宿主提供 `window._tsx.apirequest`，优先走宿主请求。
- 否则默认走 `/api` 代理到后端。
- 只有显式设置 `VUE_APP_REQUEST_MODE=mock` 时，才会回退到前端 mock。

这意味着现在开发环境默认是联调模式，不是 mock 模式。

## 8. 交接时最重要的认知

- 这是一个“前端完整、后端逐步去 mock 化”的项目。
- 当前 H2 数据库承担的是“开发态可联调数据底座”，不是正式生产数据库。
- 文档里的“后台计算逻辑”指的是服务层对原始事实数据和元数据做聚合、映射、衍生计算，不代表已经接入真实实时业务系统。
