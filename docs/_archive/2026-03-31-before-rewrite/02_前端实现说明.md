# 前端实现说明

## 1. 前端公共结构

### 应用入口

- `frontend/src/main.js` 负责挂载 Vue、路由、Element UI 和宿主生命周期。
- `frontend/src/router/index.js` 负责全部页面路由。
- `frontend/src/settings.js` 负责左侧菜单与模块导航说明。
- `frontend/src/api/pvms.js` 统一封装所有后端接口。
- `frontend/src/shared/host/bridge.js` 负责宿主请求、开发代理、mock 切换。

### 通用 UI / 技术点

- UI 组件库主用 `element-ui`
- 图表主用 `echarts`
- 地图底图主用 `@amap/amap-jsapi-loader`
- 页面主体统一挂在 `ShellLayout.vue`
- 各模块页面基本都采用“Hero + FilterBar + TabNav + 视图组件”的结构

## 3. 前端维护时最容易踩坑的点

- 路由是 `history` 模式，前端服务必须能正确 fallback 到 `index.html`。
- 开发环境默认应联调后端，不要把 `VUE_APP_REQUEST_MODE` 误改成 `mock`。
- 地图底图报错不等于你后端有问题，`jsapi-data*.amap.com` 是高德外部资源。
- 大部分页面都不是一个接口渲染全部，而是多个接口拼装；排错时先确认页面上每块 UI 对应哪个 API。
