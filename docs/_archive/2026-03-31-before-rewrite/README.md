# 文档导航

这套文档现在只保留 5 类主文档，按阅读顺序往下看即可，不需要再在一堆子目录里来回跳。

## 先看哪几类

1. [01_系统总览与环境.md](./01_系统总览与环境.md)
   作用：先搞清楚这个系统是什么、现在做到哪一步、怎么启动、主入口在哪。
2. [02_前端实现说明.md](./02_前端实现说明.md)
   作用：按模块顺序看前端页面、组件、路由、数据渲染点和接口调用关系。
3. [03_后端实现说明.md](./03_后端实现说明.md)
   作用：按模块顺序看接口、返回结构、数据表、字段来源和核心计算逻辑。
4. [04_测试与联调说明.md](./04_测试与联调说明.md)
   作用：看已经测了什么、怎么跑测试、怎么联调、常见问题怎么排查。
5. [05_审计结论与后续计划.md](./05_审计结论与后续计划.md)
   作用：看当前风险、哪些地方还在 mock、距离 release 还差什么、后续建议怎么排。

## 文档设计原则

- 一个类别只保留一个总文档。
- 每个总文档内部再按模块顺序堆起来，不再拆成每个模块一个 md。
- 你要交接时，优先发这 5 个文件，不需要把旧的散文档一起发出去。
