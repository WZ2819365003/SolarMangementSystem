# 环境准备与本地启动

## 1. 必备环境

本项目存在两套前端运行环境要求，不要混淆：

| 用途 | 版本要求 | 说明 |
| --- | --- | --- |
| 前端主应用运行/构建 | Node 14.16.0 + npm 6.14.11 | 见 `frontend/package.json` |
| Playwright 自动化 | Node 18+ | 见 `frontend/tests/playwright/package.json` |
| 后端运行 | Java 21 | 见 `backend/pom.xml` |
| 后端构建/测试 | Maven 3.x | 本地需可执行 `mvn` |

## 2. 前端环境文件

前端示例环境文件位于：

- `frontend/.env.example`

当前示例字段：

```env
VUE_APP_AMAP_KEY=your_amap_key
VUE_APP_AMAP_SECURITY_CODE=your_amap_security_code
VUE_APP_AMAP_VERSION=2.0
```

如果高德地图相关页面报错，优先检查这几个变量是否配置。

## 3. 前端依赖注意事项

前端依赖里有一个需要特别注意的本地包：

- `hs-dtk-ui`: `file:../../vpp-main/node_modules/hs-dtk-ui/hs-dtk-ui-0.1.29.tgz`

这意味着：

- 这不是一个公共 npm 依赖
- 如果本机没有对应路径，`npm install` 会失败
- 新同事接手时，需要先确认本机是否具备 `vpp-main` 依赖目录，或者改成团队统一可访问的包源

## 4. 启动前端

在仓库根目录执行：

```powershell
cd frontend
C:\Users\zhuow\tools\node-v14.16.0-win-x64\npm.cmd install --scripts-prepend-node-path=true
C:\Users\zhuow\tools\node-v14.16.0-win-x64\npm.cmd run serve --scripts-prepend-node-path=true
```

默认端口：

- `http://127.0.0.1:6618`

开发服务器配置位于：

- `frontend/vue.config.js`

关键点：

- dev server 端口固定为 `6618`
- 本地开发 `publicPath` 为 `/`
- 生产构建 `publicPath` 为 `/pvms/`

## 5. 启动后端

在仓库根目录执行：

```powershell
cd backend
mvn test
mvn spring-boot:run
```

默认端口：

- `http://127.0.0.1:8091`

基础健康检查：

- `GET http://127.0.0.1:8091/api/system/health`

配置文件位于：

- `backend/src/main/resources/application.yml`

## 6. 启动顺序建议

建议按下面顺序操作：

1. 先起后端，确认 `health` 正常
2. 再起前端
3. 打开前端页面确认菜单和路由正常
4. 如需跑 Playwright，再单独进入 `frontend/tests/playwright`

## 7. 运行 Playwright

Playwright 目录和主前端目录的 Node 版本要求不同，请单独处理。

```powershell
cd frontend/tests/playwright
npm install
npm run install:browsers
npm test
```

Playwright 配置文件：

- `frontend/tests/playwright/playwright.config.js`

当前配置特点：

- 自动拉起前端开发服务器
- 默认访问 `http://127.0.0.1:6618/dashboard/overview`
- 失败时保留 trace 和截图
- HTML 报告输出到 `frontend/tests/playwright/output/report`

## 8. 本地启动后的验证动作

前端启动后，建议手工访问以下页面确认路由都可打开：

- `/dashboard/overview`
- `/production-monitor/overview`
- `/forecast/overview`
- `/strategy/list`
- `/devices/monitor`
- `/alarms/center`

后端启动后，建议至少验证：

- `/api/system/health`
- `/api/pvms/dashboard/overview`
- `/api/pvms/forecast/meta`
- `/api/pvms/strategy/meta`

## 9. 常见启动问题

### `npm install` 失败

优先排查：

- Node 版本是否为 14.16.0
- `hs-dtk-ui` 本地 tgz 路径是否存在

### 地图相关页面报错

优先排查：

- `.env` 中的高德 key/security code 是否配置

### Playwright 起不来

优先排查：

- 是否使用了 Node 18+
- 浏览器是否执行过 `npm run install:browsers`

### 后端能起但接口不符合预期

这通常不是服务没起来，而是当前返回本来就是 mock 结构。先读 `07_Mock现状_联调策略_发布差距.md`。
