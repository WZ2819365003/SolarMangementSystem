## 4. 现在怎么排障

### 第一步：先判断是不是打到了真实后端

优先直接访问后端 `8091` 的接口。

如果 `8091` 正常但前端页面没数据，再看：
- 前端代理
- 页面状态
- 字段消费

### 第二步：确认 H2 是否真的有数据

打开：
- `http://127.0.0.1:8091/h2-console`

重点查看三组表：

- `dashboard_*`
- `pm_*`
- `sa_*`

如果 `sa_*` 为空，优先检查：
- `schema.sql`
- `data.sql`
- `application.yml` 里的 SQL 初始化配置

### 第三步：确认页面属于哪条后端链路

当前最容易混淆的就是 M02：

- 资源总览模式 -> `/api/pvms/production-monitor/*`
- 电站监控模式 -> `/api/pvms/station-tree`、`/api/pvms/station-archive/*`

如果接口前缀看错，排障方向会完全跑偏。

### 第四步：确认问题是“元数据”还是“计算结果”

举例：

- 树节点名、设备型号、地址  
  更可能是主数据问题

- KPI、可调空间、1 分钟曲线、逆变器功率  
  更可能是后端计算或拼装问题

## 5. 交接前必须补齐什么

### 文档层面

- 更新对应模块文档
- 更新交接手册
- 如果改了表结构，要写清楚：
  - 哪些是基础事实表
  - 哪些指标是后端计算
  - 哪些数据仍然是开发态造数

### 验证层面

- 写清楚跑了哪些命令
- 写清楚验了哪些接口
- 如果没有跑前端或 Playwright，要明确说明

### 代码层面

- 不要只改前端，不确认后端契约
- 不要只补数据库表，不补 service 计算说明
- 不要把应由后端承担的聚合逻辑偷偷搬回前端

## 6. 当前对维护者最有用的一句判断

如果某个页面“看起来有数据”，先不要默认它已经联到真实后端。

先确认：

1. 前端是不是走了 `/api` 代理
2. 后端 controller 是不是真实现
3. 数据来自 H2 / 后端计算，还是还停留在 mock
