# 光伏管理系统项目审视报告

## 1. 审视范围

- 审视时间: `2026-03-27`
- 审视对象:
  - 前端: `frontend`
  - 后端: `backend`
- 审视方式:
  - 静态阅读路由、页面、API 封装、mock 桥接、Spring Boot controller 与测试
  - 运行基础验证命令，不修改任何业务代码

## 2. 总结论

这个工程已经做出了一个可展示的前后端雏形，但当前真实状态更准确地说是:

- `模块 1` 已经有比较完整的展示链路
- `模块 2` 前端页面完成度高，但真实前后端链路没有完全打通
- `设备监控` 和 `告警中心` 目前本质上还是前端本地 mock 页面
- 后端目前是一个 `mock 契约后端`，不是正式业务后端
- 前端能 `build`，但代码质量门没有收口，`lint` 当前失败

也就是说，这个工程现在更接近“高保真演示版 + 局部后端契约实现”，不是“真实联调完成版”。

## 5. 当前链路是怎么走的

### 5.1 前端请求总入口

前端所有请求统一先走:

- `frontend/src/api/pvms.js`

再进入:

- `frontend/src/shared/host/bridge.js`

桥接逻辑当前是:

- 只有 `production + 有 window._tsx.apirequest` 时才走宿主请求
- 其他情况默认走 `frontend/src/shared/mock/data.js`

这意味着开发态默认不是打 Spring Boot，而是打前端本地 mock。

### 5.2 模块 1 链路

#### 已打通到后端契约的部分

- 地图: `stations-geo`
- KPI: `kpi-summary`
- 曲线: `power-curve`
- 排名: `station-ranking`
- 最近告警: `alarms/recent`
- 当前天气: `weather/current`

#### 前端额外补出来的部分

- `/pvms/adjustable-capacity/realtime`
- `/pvms/dashboard/vpp-node-status`

这两项当前在后端没有 controller，对应前端 mock:

- `frontend/src/shared/mock/data.js:1048-1085`

### 5.3 模块 2 链路

#### 已有后端契约支撑的部分

- `meta`
- `overview`
- `output`
- `dispatch`
- `weather`

#### 当前只在前端 mock 中存在的部分

- `load`
- `station tree`
- `company overview`
- `station realtime`
- `station strategy`
- `inverter realtime`

所以模块 2 现在是:

- 一部分走后端 mock 契约
- 一部分完全走前端本地 mock

### 5.4 设备监控与告警中心链路

这两个页面当前都是:

- 页面已做
- API 已定义
- 数据由 `frontend/src/shared/mock/data.js` 提供
- 后端无对应 controller

- 后端无接口

- `中高`

现状:

- 页面存在

- `中`

- 表结构和接口形态相对标准
- 但需要先补设备主数据模型

#### 告警中心后端

现状:

- 页面存在
- 纯前端 mock

难度:

- `中`

- 但如果后续要做到闭环处理、派单、确认，就会迅速变成 `中高`

### 6.2 只有展示，没有真正业务闭环

#### 导出能力

- 模块 2 页面里的导出仍然是提示语

- `frontend/src/modules/production-monitor/pages/ProductionMonitorPage.vue:369-371`

- 如果只是前端导出当前表格，难度低

- 后端 weather 仍是 mock
- 还没有真实气象源采集、站点映射、资源单元共用天气的正式规则实现

- `中高`

- 需要外部数据源

- 需要处理时间粒度和缺测问题

#### 持久化和数据库层

现状:

- 当前后端是内存 mock service
- 没有 repository、数据库、真实表结构

难度:

- `高`

原因:

- 一旦进入真实业务，就不只是补 controller，而是要重建数据层

#### 登录、租户、权限

现状:

难度:
