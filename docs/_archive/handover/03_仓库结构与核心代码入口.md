# 仓库结构与核心代码入口

## 1. 根目录结构

```text
.
├─ backend/        Spring Boot 后端
├─ frontend/       Vue 2 前端子应用
├─ docs/           所有项目文档
├─ data/           数据样例或辅助资料
├─ runtime-logs/   运行日志
├─ scripts/        辅助脚本
└─ README.md       仓库总入口
```

## 2. 前端目录怎么读

前端最重要的几个入口如下：

| 路径 | 作用 |
| --- | --- |
| `frontend/src/main.js` | 应用启动入口，处理 qiankun 挂载与独立运行 |
| `frontend/src/router/index.js` | 所有页面路由定义 |
| `frontend/src/settings.js` | 左侧菜单、应用名、描述 |
| `frontend/src/store/index.js` | 全局状态入口，主要承载运行时信息和菜单 |
| `frontend/src/api/pvms.js` | 前端统一接口清单 |
| `frontend/src/shared/host/bridge.js` | 宿主桥接层，决定请求走宿主还是前端 mock |
| `frontend/src/shared/mock/` | 前端开发态 mock 数据 |
| `frontend/src/modules/` | 业务模块页面和组件 |
| `frontend/src/layouts/ShellLayout.vue` | 整体外壳布局、侧边栏和头部 |

### `frontend/src/modules/` 下的业务模块

- `dashboard/`
- `production-monitor/`
- `forecast/`
- `strategy/`
- `devices/`
- `alarms/`
- `stations/`
- `station-monitoring/`

其中需要特别注意：

- `stations/` 并不只是一个独立模块，它还被 `production-monitor` 的“电站监控”模式复用。
- `station-monitoring/` 和 `stations/` 都是站点/资源相关能力，但承担的页面职责不同，维护时要先看调用入口，不要只看目录名。

## 3. 后端目录怎么读

后端当前结构比较统一：

| 路径 | 作用 |
| --- | --- |
| `backend/src/main/java/cn/techstar/pvms/backend/PvmsBackendApplication.java` | Spring Boot 启动类 |
| `backend/src/main/java/cn/techstar/pvms/backend/common/ApiResponse.java` | 统一响应体 |
| `backend/src/main/java/cn/techstar/pvms/backend/module/` | 业务模块 |
| `backend/src/main/resources/application.yml` | 应用配置 |
| `backend/src/test/java/.../module/` | 按模块组织的接口测试 |

### `backend/module/` 下的模块

- `adjustable`
- `dashboard`
- `devicealarm`
- `forecast`
- `productionmonitor`
- `stationarchive`
- `stations`
- `strategy`
- `system`

每个模块当前大多遵循类似结构：

```text
module/<module-name>/
├─ controller/
└─ service/
```

现阶段最常见的 service 命名是 `*MockService`，这说明它们提供的是演示/联调用的数据和简化逻辑，而不是完整生产服务。

## 4. 建议的阅读顺序

### 想看页面怎么组织

按这个顺序读：

1. `frontend/src/settings.js`
2. `frontend/src/router/index.js`
3. `frontend/src/layouts/ShellLayout.vue`
4. 目标模块的 `pages/*.vue`
5. 目标模块的 `components/views/*.vue`

### 想看请求怎么走

按这个顺序读：

1. 页面组件里的接口调用
2. `frontend/src/api/pvms.js`
3. `frontend/src/shared/host/bridge.js`
4. `frontend/src/shared/mock/*.js` 或后端 Controller

### 想看后端接口怎么组织

按这个顺序读：

1. 对应模块 `controller/*Controller.java`
2. 对应模块 `service/*Service.java` 或 `*MockService.java`
3. 对应测试 `*ControllerTest.java`

## 5. 新增功能时应该改哪里

### 新增前端页面

通常至少要动这些地方：

- `frontend/src/router/index.js`
- `frontend/src/settings.js`（如果要进菜单）
- `frontend/src/modules/<module>/pages/`
- `frontend/src/modules/<module>/components/`
- `frontend/src/api/pvms.js`

### 新增后端接口

通常至少要动这些地方：

- `backend/src/main/java/.../module/<module>/controller/`
- `backend/src/main/java/.../module/<module>/service/`
- `backend/src/test/java/.../module/<module>/controller/`

## 6. 哪些目录是生成物或运行产物

以下目录通常不是手写业务代码：

- `frontend/dist/`
- `frontend/node_modules/`
- `frontend/tests/playwright/output/`
- `frontend/tests/playwright/test-results/`
- `backend/target/`
- `runtime-logs/`

维护时不要把它们误当成需要长期维护的源码目录。
