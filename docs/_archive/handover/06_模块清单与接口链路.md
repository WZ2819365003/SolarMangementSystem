# 模块清单与接口链路

## 1. 总体说明

本项目的代码链路可以简化成下面这条主线：

```text
前端路由 -> 页面组件 -> api/pvms.js -> bridge.js -> 前端 mock 或后端接口 -> 后端 Controller -> MockService
```

理解这条链路之后，再看任何一个模块都会清晰很多。

## 4. 关键模块链路详解

### 综合监控中心

链路：

```text
OverviewPage.vue
  -> api/pvms.js 中 dashboard / alarms / weather 相关方法
  -> bridge.js
  -> 前端 dashboard mock 或后端 dashboard / alarms / weather controller
```

特点：

- 是典型聚合页
- 一个页面会同时拉多组接口
- 地图筛选、站点聚焦、KPI、天气、告警会联动刷新

### 生产监控

链路分成两条：

#### 资源总览模式

```text
ProductionMonitorPage.vue
  -> fetchProductionMonitorMeta
  -> fetchProductionMonitorOverview / Output / Load / Dispatch
  -> 后端 productionmonitor 模块
```

#### 电站监控模式

```text
ProductionMonitorPage.vue
  -> StationTreePanel / StationDataPanel
  -> fetchStationTree
  -> fetchStationArchiveCompanyOverview / Realtime / Strategy / InverterRealtime
  -> 后端 stationarchive 模块
```

这意味着：

- `production-monitor` 不是一个完全独立的模块
- 它部分复用了 `stations` 目录里的组件和 `stationarchive` 后端接口

### 预测与分析

链路：

```text
ForecastPage.vue
  -> 根据 route.meta.viewKey 决定加载 overview / adjustable / accuracy
  -> overview 视图会并发拉 overview + comparison + deviation-heatmap
  -> adjustable / accuracy 视图分别单独拉接口
```

### 策略管理

链路：

```text
StrategyPage.vue
  -> 先拉 meta
  -> list / config / revenue 视图各自拉自己的接口
```

子视图职责：

- `StrategyListView.vue`：列表、筛选、批量提交、批量删除
- `StrategyConfigView.vue`：树选择、单个/批量配置、收益模拟、创建、提交
- `StrategyRevenueView.vue`：收益 KPI、趋势图、占比图、策略对比

## 5. 后端接口前缀与模块对应关系

| 后端前缀 | 模块目录 | 说明 |
| --- | --- | --- |
| `/api/pvms/dashboard` | `backend/module/dashboard` | 首页地图、KPI、曲线、排名等 |
| `/api/pvms/alarms` | `backend/module/dashboard` + `backend/module/devicealarm` | 最近告警和告警中心入口分属不同 controller |
| `/api/pvms/weather` | `backend/module/dashboard` | 天气信息 |
| `/api/pvms/production-monitor` | `backend/module/productionmonitor` | 生产监控资源总览接口 |
| `/api/pvms/station-tree` / `/api/pvms/station-archive/*` | `backend/module/stationarchive` | 电站树、公司总览、单站实时、逆变器实时等 |
| `/api/pvms/resource-units` | `backend/module/stations` | 资源单元列表和详情 |
| `/api/pvms/forecast` | `backend/module/forecast` | 预测与分析 |
| `/api/pvms/strategy` | `backend/module/strategy` | 策略管理 |
| `/api/pvms/devices/monitor` | `backend/module/devicealarm` | 设备监控 |
| `/api/system` | `backend/module/system` | 健康检查 |
