# M04 策略管理 H2 数据与计算说明

## 1. 这次改造做了什么

这次 M04 不是“补一个接口”，而是把 `strategy` 模块从缺失态推进到了可联调态。

最终形态是：

- 前端有真实菜单、真实路由、真实页面
- 后端有真实 controller / service / repository
- H2 中有 `sg_*` 策略元数据表
- 收益和模拟结果不再由前端 mock 直接拼
- 开发环境已经可以走 `/api/pvms/strategy/*`

## 2. 数据边界怎么理解

### 2.1 什么是元数据

在 `strategy` 里，下面这些属于开发态元数据：

- 策略主表
- 分时段动作
- 执行日志
- 电价模板
- 收益日表
- 模拟快照

这些数据当前保存在 H2：

- `sg_strategy`
- `sg_strategy_period`
- `sg_execution_log`
- `sg_price_period`
- `sg_revenue_daily`
- `sg_strategy_snapshot`

### 2.2 什么是系统内事实数据

策略收益不是完全靠 `sg_*` 算出来的，它会读系统内其他模块的数据：

- `sa_station`
- `sa_station_curve_15m`
- `fc_prediction_series_15m`

当前后端计算会用到：

- 电站负荷
- 电站实际光伏出力
- 日前预测出力
- 预测上下界
- 站点容量和归属

### 2.3 什么必须留在后端计算

即使未来接真实系统，下面这些也建议继续保留在后端，不要挪回前端：

- 策略 KPI 汇总
- 树结构统计
- 单策略模拟收益
- 置信区间
- 批量模拟总收益
- 收益趋势和类型拆分
- 策略对比差值

原因很简单：

- 这些都依赖多表聚合
- 公式口径需要统一
- 前端不应该持有业务结算逻辑

## 6. 公式和口径当前怎么来的

当前模拟收益由这三块组成：

- `peakSaving`
- `responseReward`
- `penalty`

这些值来自：

- 站点负荷与实际出力差
- 日前预测与实际出力偏差
- 电价时段价格
- 策略类型系数

当前公式是开发态口径，不是生产结算口径。它的作用是：

- 跑通链路
- 固化数据边界
- 让前后端围绕真实契约开发

后续对接真实系统时，应优先替换公式和数据源，而不是推翻接口。

## 7. 后续怎么替换成真实系统

### 7.1 电价

把 `sg_price_period` 换成真实电价服务或数据库表。

### 7.2 收益

把 `sg_revenue_daily` 换成真实收益结算或市场结算系统。

### 7.3 执行日志

把 `sg_execution_log` 换成真实策略执行链路审计。

### 7.4 模拟输入

如果未来预测服务、调度服务、设备服务独立部署：

- 保留 `StrategySimulationService`
- 把内部 repository 取数替换成服务调用或新的 repository
