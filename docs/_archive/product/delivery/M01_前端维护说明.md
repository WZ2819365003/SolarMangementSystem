# M01 前端维护说明

## 1. 技术栈与运行方式

前端技术栈：

- Vue 2
- Vue Router 3
- Vuex 3
- Element UI 2
- hs-dtk-ui 0.1.29
- ECharts 5
- 高德地图 JS API Loader

本地运行环境：

- Node `14.16.0`
- npm `6.14.11`

Playwright 测试环境：

- 独立于业务前端
- 位于 `frontend/tests/playwright`
- 需要 `Node 18+`

## 4. 请求链路

### 4.1 API 封装入口

- `frontend/src/api/pvms.js`

这个文件只负责定义前端要调用的接口方法，不负责真正发请求。

### 4.2 桥接层

- `frontend/src/shared/host/bridge.js`

当前行为：

- 如果是生产态且宿主存在 `window._tsx.apirequest`
  - 走宿主请求
- 其他情况
  - 走 `frontend/src/shared/mock/data.js`

这意味着本地开发时：

- 模块 1 页面默认使用前端 mock
- 不会直接请求 `http://127.0.0.1:8091`

如果后续需要联调本地后端，要改这里。

## 7. 当前要特别注意的维护点

### 7.1 前端 mock 与后端接口是两套并行体系

当前模块 1 前端展示用的是：

- `frontend/src/shared/mock/data.js`

后端接口在：

- `backend/src/main/java/.../DashboardMockService.java`

它们数据结构是一致的，但不是同一条运行链路。

### 7.2 修改 GIS 交互时要同时关注 3 个点

- `OverviewPage.vue`
  - 是否触发 `mapFocusToken`
- `DashboardMapCard.vue`
  - 是否改变了总览态 / 聚焦态切换
- `dashboard-overview.spec.js`
  - 是否仍然通过地图缩放与 reset 回归校验

### 7.3 现在的首页布局已经过 Playwright 校验

不要只靠肉眼改首页布局，改完至少跑：

- `frontend/tests/playwright/specs/dashboard-overview.spec.js`

## 8. 建议的后续维护方式

如果后续继续扩模块 1，建议按这个顺序：

1. 先改 `doc/` 里的设计稿
2. 改 `OverviewPage.vue` 的数据编排
3. 改单个组件
4. 补 Playwright 回归断言
5. 最后再更新 `doc/finished/`
