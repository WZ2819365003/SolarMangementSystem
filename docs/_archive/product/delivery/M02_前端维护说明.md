# M02 前端维护说明

## 1. 技术栈与运行方式

前端技术栈：

- Vue 2
- Vue Router 3
- Vuex 3
- Element UI 2
- hs-dtk-ui 0.1.29
- ECharts 5

本地运行环境：

- Node `14.16.0`
- npm `6.14.11`

Playwright 测试目录：

- `frontend/tests/playwright`

## 2. 路由结构

模块 2 当前正式路由：

- `/production-monitor/overview`
- `/production-monitor/output`
- `/production-monitor/dispatch`
- `/production-monitor/weather`

旧路由兼容：

- `/station`
- `/station/:id`

当前只保留重定向，不再承担模块 2 主链路语义。

API 封装文件：

- `frontend/src/api/pvms.js`

- `fetchProductionMonitorMeta`

- `fetchProductionMonitorWeather`

- `frontend/src/shared/host/bridge.js`

开发态默认行为：

- 模块 2 页面优先走前端 mock
- 不直接请求本地 Spring Boot

mock 文件：

- `frontend/src/shared/mock/station-monitoring.js`

## 5. 前端数据建模规则

模块 2 前端 mock 已经固定以下语义：

- 同一资源单元共享天气
- 同一资源单元共享总出力趋势
- 成员电站实时出力由总出力按权重和状态系数分摊
- 成员电站距离过远时，不应继续塞进同一个资源单元

这个规则不要在单个页面里临时打破，否则页面解释会混乱。

## 6. 当前维护重点

### 6.1 如果改横向导航

同时要检查：

- `ProductionMonitorPage.vue`
- `frontend/src/router/index.js`
- `frontend/src/settings.js`
- `station-monitoring.spec.js`

### 6.2 如果改共享筛选

同时要检查：

- `ProductionMonitorFilterBar.vue`
- `ProductionMonitorPage.vue`
- 4 个视图组件是否仍然共享同一套 query

### 6.3 如果改天气或出力逻辑

同时要检查：

- `frontend/src/shared/mock/station-monitoring.js`
- `WeatherBriefCard.vue`
- `WeatherTrendPanel.vue`
- `PowerCurvePanel.vue`
- 成员电站摘要与贡献排行是否仍然共用同一套天气解释

## 7. Playwright 验证

当前主要回归文件：

- `frontend/tests/playwright/specs/station-monitoring.spec.js`
- `frontend/tests/playwright/specs/layout-smoke.spec.js`

每次改模块 2，建议至少跑：

1. `npm.cmd test -- station-monitoring.spec.js`
2. `npm.cmd test -- layout-smoke.spec.js`

## 8. 后续扩展建议

后续继续扩模块 2 时，建议优先扩：

1. 调度指令明细
2. 资源单元响应追踪
3. 天气采集对接
4. 预测偏差归因

不建议直接回退到设备监控页思路。
