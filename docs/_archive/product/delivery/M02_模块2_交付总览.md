# M02 模块 2 交付总览

## 1. 模块范围

模块 2 对应 `生产监控层`，当前交付的是虚拟电厂聚合商视角下的光伏生产监控模块。

本轮完成内容包括：

- 前端模块壳子重构
- 左侧一级菜单接入
- 顶部横向导航切换 4 个生产视图
- 前端共享筛选状态
- 前端 mock 数据重建
- Spring Boot mock 接口重建
- Playwright 布局与交互回归
- 后端 MockMvc 测试
- 模块设计文档与维护文档

## 2. 当前功能

### 2.1 前端功能

前端当前可用视图：

- `/production-monitor/overview`
- `/production-monitor/output`
- `/production-monitor/dispatch`
- `/production-monitor/weather`

每个视图都围绕 `聚合资源单元` 展示，不再采用旧的 `/station` 列表 + 详情页链路。

### 2.2 后端功能

当前后端接口：

- `/api/system/health`
- `/api/pvms/production-monitor/meta`
- `/api/pvms/production-monitor/overview`
- `/api/pvms/production-monitor/output`
- `/api/pvms/production-monitor/dispatch`
- `/api/pvms/production-monitor/weather`

## 3. 当前业务规则

模块 2 最重要的建模规则是：

- 同一个资源单元中的电站地理位置接近
- 因此共用同一套天气剖面
- 也共用同一条资源单元总出力趋势
- 各电站只按权重和状态系数分摊出力比例
- 如果站点距离过远，应拆分为新的资源单元

这条规则已经同时落在：

- 前端 mock
- 后端 mock service
- 页面说明与设计文档

## 4. 交付边界

这轮明确没有做：

- 深设备页
- 单站设备监控
- 一次系统图
- 逆变器、汇流箱、箱变等设备级页面
- 真实天气采集
- 真实数据库和真实业务服务

模块 2 当前定位是：

- 结构清晰的聚合商生产监控模块
- 不是单站设备运维平台

## 6. 一定要知道的现状

虽然模块 2 的后端接口已经完成，但前端开发态默认仍然优先走前端 mock，而不是直接请求 Spring Boot。

也就是说：

- 当前看到的前端页面默认由前端 mock 支撑
- 后端接口主要承担契约、测试和后续联调基线的角色

## 8. 后续建议

如果继续扩展模块 2，建议优先按以下顺序：

1. 接真实天气采集
2. 接真实资源单元出力数据
3. 增加调度指令明细
4. 增加响应追踪
5. 增加预测偏差归因
