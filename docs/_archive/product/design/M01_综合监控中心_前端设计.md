# M01 综合监控中心（前端设计）

> 模块定位：光伏管理系统首页驾驶舱
> 用户角色：管理层、调度员、值班运维
> 技术栈：Vue 2 + Element UI + hs-dtk-ui + ECharts 5 + 高德地图
> 路由：`/dashboard/overview`
> 版本：V4.1（2026-03-27）
> 状态：✅ 已完成

---

## 1. 设计结论

M01 作为系统首页，提供一屏总览能力。不做深度钻取，而是通过聚合数据和快捷跳转，帮助管理层和调度员快速掌握全局态势。

**核心职责**：
- 一屏展示全系统光伏运营态势
- 聚合展示 VPP 节点可调能力
- 提供向 M02 生产监控层的快捷钻取

---

## 2. 页面结构

采用两层结构：

- **上层**：GIS 电站总览（含底部运行洞察区）+ 核心 KPI 看板（分组展示）
- **下层**：计划 vs 实际功率曲线（含可调区间带）+ 发电排名 + 实时告警

> ⚠️ V4.0 变更：已移除天气卡片，KPI 看板直接与地图卡片等高对齐。

---

## 3. 已落地内容

> **V4.1 实施说明**：以下各小节所描述的功能均已在前端代码中完成开发与集成，组件文件已就位，接口已对接。V4.0 提出的移除天气卡片变更已落地实施，KPI 看板已直接与地图卡片等高对齐。

### 3.1 GIS 电站总览

- 地图展示电站点位、状态和选中态
- 支持状态、区域、容量筛选
- 支持电站详情浮层
- 底部运行洞察区（三栏）：
  - **重点关注电站**：故障/告警/离线电站优先展示，点击联动地图聚焦
  - **VPP 节点状态摘要**：可调总量、今日已响应次数、累计响应电量、策略执行率
  - **区域态势**：按区域聚合展示电站数/容量/在线率

### 3.2 核心 KPI 看板（分组展示）

KPI 指标按业务分类展示，每组有标题标签：

| 分组 | 指标 |
|---|---|
| 发电概况 | 总装机容量、实时总功率、当日发电量、当日收益 |
| 运行质量 | 等效利用小时、在线率、碳减排量、偏差率 |
| VPP 可调能力 | 可上调容量、可下调容量 |

### 3.3 功率曲线

- 实际功率、日计划、超短期预测、辐照度
- **叠加可调能力区间带**（半透明面积，展示可上调/可下调范围）

### 3.4 发电排名

- 按日发电量、等效小时、PR 值切换排序

### 3.5 实时告警

- 告警等级汇总、滚动列表和详情抽屉

## 5. 数据策略

### 5.1 已有接口

| 接口 | 说明 |
|---|---|
| GET /api/pvms/dashboard/meta | 全局概要（KPI + 统计） |
| GET /api/pvms/dashboard/stations | 电站列表（含经纬度） |
| GET /api/pvms/dashboard/power-curve | 功率曲线 |
| GET /api/pvms/dashboard/alarms | 告警列表 |
| GET /api/pvms/dashboard/ranking | 发电排名 |
| GET /api/pvms/adjustable-capacity/realtime | 聚合可调能力 |
| GET /api/pvms/dashboard/vpp-node-status | VPP 节点状态摘要 |

## 6. 联动跳转

| 触发 | 目标 | 说明 |
|---|---|---|
| 地图点击电站 | M02 电站监控 | `$router.push('/production-monitor/station?nodeType=station&nodeId=xxx')` |
| 重点关注电站点击 | M02 电站监控 | 同上，定位到该电站 |
| 点击告警 | M06 告警管理 | 跳转告警详情 |

> ⚠️ V4.0 变更：跳转目标从原 M03 `/stations/archive` 改为合并后的 M02 `/production-monitor/station`。

---

## 7. 样式要求

- 深蓝底板 + 柔和描边，延续宿主系统主题
- 蓝、青、绿、橙、红作为状态色
- KPI 分组标题使用 teal 竖条装饰
- 洞察区只做轻动画，不做重交互
- 所有组件使用 `--pvms-*` 设计令牌
