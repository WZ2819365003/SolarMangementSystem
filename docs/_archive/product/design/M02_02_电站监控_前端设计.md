# M02-02 电站监控（前端设计）

> 修订日期: 2026-03-27
> 版本: V4.1（从原 M02 生产监控层拆分）
> 模块定位: 面向单站的钻取监控视图（左侧树 + 右侧数据面板）
> 技术基线: `Vue 2 + Element UI + hs-dtk-ui + ECharts 5 + 宿主主题变量`
> 实现状态: ✅ 前端已完成

---

## 1. 模块定位

电站监控是生产监控层的单站钻取视角。通过左侧三级树选择节点，右侧面板展示对应层级的数据。它与 M02-01 资源总览构成上下钻取关系：总览看全局，电站监控看单站细节。

---

## 2. 路由

| 路由 | 说明 |
|---|---|
| `/production-monitor/station` | 电站监控主页面 |

**URL Query 参数**

| 参数 | 示例 | 说明 |
|---|---|---|
| `nodeType` | `company` / `station` / `inverter` | 当前选中节点类型 |
| `nodeId` | `com-001` / `SZ-001` / `inv-001` | 当前选中节点 ID |
| `tab` | `adjustable` | 电站层级当前数据 Tab |

页面加载时若 URL 含 `nodeId`，自动展开树并选中对应节点。

## 4. 左侧树面板

### 4.1 三级树结构

```
Level 1: 公司（代理用户）
  └── Level 2: 电站（= VPP 资源节点）
        └── Level 3: 逆变器（子设备）
```

### 4.2 树节点数据模型

```javascript
{
  id: 'com-001',
  label: '深圳市悦美颐华物业管理有限公司',
  nodeType: 'company',       // 'company' | 'station' | 'inverter'
  statusTag: '代理用户',
  statusColor: '#409eff',
  children: [
    {
      id: 'SZ-001',
      label: '深圳湾科创园 A 站',
      nodeType: 'station',
      statusTag: '电站',
      statusColor: '#67c23a',
      extra: {
        capacityKwp: 1800,
        status: 'normal',     // normal | fault | offline | warning
        adjustableKw: 420,
        pvOutputKw: 1380,
        loadKw: 1800
      },
      children: [
        { id: 'inv-001', label: 'INV-01 (600kW)', nodeType: 'inverter', statusTag: '逆变器', statusColor: '#e6a23c' }
      ]
    }
  ]
}
```

### 4.3 树面板功能

| # | 功能 | 描述 |
|---|---|---|
| 1 | 状态筛选 Tab | 全部 / 在线 / 离线 / 故障，过滤树节点 |
| 2 | 搜索框 | 300ms 防抖，匹配公司名/电站名/逆变器名 |
| 3 | 三级树 | `el-tree` 组件，节点前显示类型标签（公司=蓝、电站=绿、逆变器=橙） |
| 4 | 节点状态指示 | 电站节点显示小圆点（绿=正常 / 红=故障 / 灰=离线 / 橙=告警） |
| 5 | URL 定位 | URL 含 `nodeId` 时自动展开并选中 |

## 5. 右侧数据面板 — 公司层级（nodeType = company）

### 5.1 KPI 卡片组（3x3 九宫格）

| 行 | 卡片 1 | 卡片 2 | 卡片 3 |
|---|---|---|---|
| 第 1 行 | 总装机(MWp) | 实时功率(MW) | 当日发电(MWh) |
| 第 2 行 | 总可调空间(MW) | 光伏收益-今日(元) | 调控收益-今日(元) |
| 第 3 行 | 本月光伏收益(元) | 本月调控收益(元) | 在线率(%) |

### 5.2 Tab 页签

#### Tab 1: 下属电站

电站列表表格，展示该公司下所有电站的运行概况。

| 列 | 说明 |
|---|---|
| 电站名称 | 可点击，点击后树展开到该电站并切换右侧为电站层级视图 |
| 装机容量 | kWp |
| 实时功率 | kW |
| 负荷 | kW |
| 可调空间 | kW |
| 状态 | 正常/告警/故障/离线 |

**表格样式要求**：深色主题下不使用白色背景。选中行使用 `rgba(6, 162, 153, 0.15)` 半透明 teal 背景。

#### Tab 2: 策略查看

策略表格，展示该公司下所有电站的当前策略。

| 列 | 说明 |
|---|---|
| 电站名称 | - |
| 策略名称 | - |
| 业务场景 | 削峰/填谷/正常运行 |
| 状态 | 执行中/待执行/已结束 |
| 目标功率 | kW |
| 操作 | 「查看」按钮，点击打开策略详情抽屉 |

### 5.3 策略详情抽屉（Drawer）

从右侧滑出，展示单个电站的策略详情。

| 区域 | 内容 |
|---|---|
| 基本信息 | 电站名称、策略名称、业务场景、执行状态、时间范围 |
| 分时电价 | 9 个时段的电价配置表，每个时段标注：峰/平/谷 + 对应策略动作（削峰/填谷/正常运行） |
| 执行概要 | 今日执行次数、成功率、累计响应电量、预估收益 |

| # | 控件 | 功能 |

### 6.5 数据 Tab（5 个）

| Tab Key | 标签 | 图表类型 | 核心内容 | Y 轴单位 |
|---|---|---|---|---|
| `adjustable` | 可调空间 | 面积+折线 | **核心 Tab**：可调空间曲线（= 负荷 - 出力）+ 可上调/可下调区间 + 负荷线 + 出力线 | kW |
| `pv-output` | 光伏出力 | 折线图 | 总出力 + 各逆变器分色叠加 + 15min 平均 | kW |
| `load` | 负荷监控 | 折线图 | 电站用电总负荷 + 分项负荷 | kW |
| `forecast` | 出力预测 | 折线图 | 日前预测 vs 超短期预测 vs 实际出力，偏差区域高亮 | kW |
| `strategy` | 策略查看 | 卡片+表格 | 只读：当前策略信息 + 执行记录（编辑在 M04） | -- |

图表区域填充剩余空间，内容超出时可滚动。

#### 可调空间 Tab（核心 Tab）

```
可调空间 = 电站负荷 - 光伏出力
```

- 实线 1：电站负荷曲线（蓝色）
- 实线 2：光伏出力曲线（绿色）
- 面积区：可调空间填充（正值=可下调，teal 色）
- 虚线 1：可上调容量上限
- 虚线 2：可下调容量下限
- 辅助 KPI（图表上方）：当前可调空间、可上调峰值、可下调峰值、可调利用率

#### 策略查看 Tab（只读）

| 区域 | 内容 |
|---|---|
| 当前策略表格 | 策略名称、业务场景、状态（执行中/待执行/已结束）、目标功率、时间范围 |
| 执行记录日志 | 最近执行记录：时间、目标、实际、偏差、状态 |
| 详情抽屉 | 点击「查看」按钮从右侧滑出抽屉，展示策略完整信息（基本信息 + 分时电价 + 执行概要） |

分时电价配置共 9 个时段，每个时段标注：
- 电价类型：峰 / 平 / 谷
- 策略动作：削峰 / 填谷 / 正常运行

## 10. 数据流

```
页面初始化:
  created() → fetchStationTree()
    → 若 URL 含 nodeId → expandToNode() → 加载对应层级数据

树节点点击:
  @node-click(node) →
    更新 URL query: ?nodeType=xxx&nodeId=xxx
    if (company)  → loadCompanyOverview(node.id)
    if (station)  → loadStationData(node.id, currentTab, currentDate)
    if (inverter) → loadInverterData(node.id, currentDate)

电站数据 Tab 切换:
  @tab-change(tabKey) → loadStationData(stationId, tabKey, currentDate)
  注意：KPI 卡片和本月 VPP 统计不重新加载

日期切换:
  @date-change(date) → loadStationData(stationId, currentTab, date)

自动刷新:
  setInterval(() => loadStationData(...), refreshInterval * 1000)
  beforeDestroy 时 clearInterval
```

## 11. 后端接口（API）

| # | 接口 | 参数 | 说明 |
|---|---|---|---|
| 1 | GET /api/pvms/station-tree | status | 三级树（公司 → 电站 → 逆变器） |
| 2 | GET /api/pvms/companies/:id/overview | - | 公司汇总 KPI（9 项）+ 电站列表 + 策略列表 |
| 3 | GET /api/pvms/stations/:id/adjustable | date, granularity | 可调空间数据（负荷/出力/可调曲线） |
| 4 | GET /api/pvms/stations/:id/pv-output | date, granularity | 光伏出力（总出力 + 逆变器分色） |
| 5 | GET /api/pvms/stations/:id/load | date, granularity | 负荷数据（总负荷 + 分项） |
| 6 | GET /api/pvms/stations/:id/forecast | date | 出力预测（日前 + 超短期 vs 实际） |
| 7 | GET /api/pvms/stations/:id/strategy | - | 当前策略信息 + 执行记录（只读） |
| 8 | GET /api/pvms/stations/:id/strategy/:strategyId | - | 策略详情（分时电价 + 执行概要） |
| 9 | GET /api/pvms/inverters/:id/realtime | date | 逆变器实时运行数据 |

当前前端使用 mock 数据，接口格式已按上述规范定义。

## 12. 样式规范

- **主题**：深蓝色暗色主题，使用 `--pvms-*` 系列 CSS 变量
- **布局**：flex 布局，左树 320px 固定，右面板 flex:1
- **树面板**：背景 `rgba(6, 20, 48, 0.96)`，填充到底部，超出滚动
- **右侧面板**：固定高度 `calc(100vh - 160px)`，`overflow-y: auto`
- **树节点标签**：`el-tag` mini 尺寸，公司=蓝 `#409eff`、电站=绿 `#67c23a`、逆变器=橙 `#e6a23c`
- **图表最小高度**：520px
- **表格**：深色主题下不使用白色背景色，选中行 `rgba(6, 162, 153, 0.15)` teal 半透明背景

```less
/deep/ .el-table__body tr.current-row > td {
  background: rgba(6, 162, 153, 0.15) !important;
  color: var(--pvms-text-primary) !important;
}
/deep/ .el-table__body tr.hover-row > td {
  background: rgba(255, 255, 255, 0.04) !important;
}
```

- **数据 Tab**：`el-tabs` card 类型
- **工具栏按钮**：`el-button` primary/plain mini
- **抽屉**：从右侧滑出，宽度 480px
