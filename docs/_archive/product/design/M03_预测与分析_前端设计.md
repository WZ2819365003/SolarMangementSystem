# M03 预测与分析（前端设计）

> 模块定位：光伏出力预测与可调能力分析
> 用户角色：调度员、分析师、管理层
> 技术栈：Vue 2 + Element UI + hs-dtk-ui + ECharts 5 + LESS
> 路由：`/forecast/overview`、`/forecast/adjustable`、`/forecast/accuracy`
> 版本：V4.1（2026-03-27）
> 状态：📋 设计完成，待施工

---

## 1. 设计结论

M03 作为预测与分析模块，提供光伏出力预测、可调能力分析和精度评估三大能力。本模块不执行策略，而是为下游模块提供决策依据。

**核心职责**：
- 提供日前预测和超短期预测的出力对比分析
- 计算并展示可调能力预测，上报 VPP 平台
- 评估预测精度，持续优化预测模型

**核心业务链路**：

```
日前预测 → 超短期修正 → 实际出力 → 偏差分析 → 精度评估
预测出力 + 负荷预测 → 可调能力预测 → 上报VPP
```

**跨模块关系**：
- M03 预测数据 → M04（策略管理）收益模拟的输入
- M03 可调能力 → VPP 平台上报
- M02 电站监控的"出力预测"Tab 使用 M03 的简化数据

## 5. 预测总览（Tab 1）

路由：`/forecast/overview`

回答的问题：预测结果与实际出力的偏差如何？哪些电站偏差较大？

### 5.1 KPI 卡片组

| # | 指标 | 单位 | 说明 |
|---|---|---|---|
| 1 | 日前预测精度 | % | 当前选定时段的日前预测准确率 |
| 2 | 超短期预测精度 | % | 当前选定时段的超短期预测准确率 |
| 3 | 当日预测偏差 | MW | 预测与实际的绝对偏差 |
| 4 | 预测合格率 | % | 偏差在阈值内的时段占比 |

### 5.2 预测对比曲线

- 三线同图：日前预测（蓝色实线）、超短期预测（青色实线）、实际出力（绿色实线）
- X 轴：时间轴（15min 粒度）
- Y 轴：功率（MW）
- 偏差区域：预测与实际之间高亮填充（正偏差橙色、负偏差红色半透明）
- 图表支持 `dataZoom` 时间轴拖拽缩放

### 5.3 偏差热力图

- 矩阵结构：X 轴 24h（每小时一格）× Y 轴 电站列表
- 色阶：偏差越大颜色越深（绿→黄→橙→红）
- 交互：hover 显示 tooltip（电站名、时段、偏差值、偏差率）
- 点击单元格：联动预测对比曲线，定位到对应电站和时段

### 5.4 电站预测明细表

| 列 | 说明 |
|---|---|
| 电站名称 | 电站名称 |
| 预测出力(MW) | 当前时段预测功率 |
| 实际出力(MW) | 当前时段实际功率 |
| 偏差(MW) | 预测 - 实际 |
| 预测精度(%) | 1 - |偏差| / 实际 |

支持按偏差/精度排序，行点击联动热力图高亮。

## 6. 可调能力分析（Tab 2）

路由：`/forecast/adjustable`

回答的问题：未来 24 小时各电站的可调能力如何？何时可用于调度？

### 6.1 KPI 卡片组

| # | 指标 | 单位 | 说明 |
|---|---|---|---|
| 1 | 总可调容量 | MW | 当前全系统可调总量 |
| 2 | 可上调容量 | MW | 可增加出力的总空间 |
| 3 | 可下调容量 | MW | 可减少出力的总空间 |
| 4 | 24h 最大可调 | MW | 未来 24h 内可调峰值 |

### 6.2 可调能力预测曲线

- 面积图 + 折线：
  - 虚线上限：可上调上限（蓝色虚线）
  - 虚线下限：可下调下限（橙色虚线）
  - 面积填充：预测可调范围（teal 半透明区域）
  - 实线：预测可调容量（teal 实线）
- X 轴：未来 24h 时间轴
- Y 轴：容量（MW）
- 辅助线：当前时刻竖线标记

### 6.3 电站可调能力明细表

| 列 | 说明 |
|---|---|
| 电站名称 | 电站名称 |
| 当前可调容量(MW) | 当前时刻可调空间 |
| 未来 4h 预测可调(MW) | 未来 4 小时预测平均可调 |
| 最大可上调(MW) | 24h 内可上调峰值 |
| 最大可下调(MW) | 24h 内可下调峰值 |

### 6.4 VPP 节点可调时间线

- 甘特图样式：X 轴为未来 24h，Y 轴为各电站
- 每个电站一行，按时间段标记可调度状态：
  - 绿色块：可调度（可调容量 > 阈值）
  - 灰色块：不可调度（可调容量不足）
  - 橙色块：受限调度（接近限值）
- 交互：hover 显示该时段详细可调数据

## 7. 精度评估（Tab 3）

路由：`/forecast/accuracy`

回答的问题：预测模型长期表现如何？哪些电站精度较差需要关注？

### 7.1 KPI 卡片组

| # | 指标 | 单位 | 说明 |
|---|---|---|---|
| 1 | MAE | MW | 平均绝对误差 |
| 2 | RMSE | MW | 均方根误差 |
| 3 | 合格率 | % | 偏差在允许范围内的时段占比 |
| 4 | 本月平均精度 | % | 当月平均预测精度 |

### 7.2 精度趋势曲线

- 折线图：过去 30 天每日精度值
- 辅助线：合格阈值线（红色虚线，如 85%）
- X 轴：日期
- Y 轴：精度（%）
- 交互：点击某日，下方表格联动显示该日明细

### 7.3 偏差分布直方图

- 直方图：X 轴为偏差区间（如 0-1MW, 1-2MW, ...），Y 轴为频次
- 叠加正态分布拟合曲线
- 颜色：偏差小=绿色，偏差大=红色渐变

### 7.4 电站精度排名

- 水平条形图：各电站按预测精度从高到低排列
- 颜色编码：精度 >= 90% 绿色，80-90% 橙色，< 80% 红色
- 交互：点击电站跳转 M02 该电站的出力预测 Tab

### 7.5 月度对比表

| 列 | 说明 |
|---|---|
| 月份 | YYYY-MM |
| MAE(MW) | 该月平均绝对误差 |
| RMSE(MW) | 该月均方根误差 |
| 预测精度(%) | 该月平均精度 |
| 环比变化 | 较上月提升/下降百分点，绿色=提升、红色=下降 |

## 10. 数据流

```
页面初始化:
  created() →
    fetchForecastMeta()           // 获取区域列表、电站列表、默认选中
    loadTabData('overview')       // 加载默认 Tab 数据

Tab 切换:
  @tab-change(tabKey) →
    更新路由: $router.push(`/forecast/${tabKey}`)
    loadTabData(tabKey, filters)

筛选变更:
  @filter-change(filters) →
    loadTabData(currentTab, filters)

Tab 数据加载:
  loadTabData(tabKey, filters) →
    if (overview)    → fetchOverviewData(filters) + fetchComparisonData(filters) + fetchDeviationHeatmap(filters)
    if (adjustable)  → fetchAdjustableData(filters)
    if (accuracy)    → fetchAccuracyData(filters)

联动交互:
  热力图单元格点击 → 更新筛选条件（电站+时段），刷新预测对比曲线
  电站精度排名点击 → $router.push('/production-monitor/station?nodeType=station&nodeId=xxx&tab=forecast')
  精度趋势日期点击 → 月度对比表滚动到对应行高亮
```

## 13. 跨模块集成

### 13.1 M03 → M04（策略管理）

M03 的预测数据作为 M04 收益模拟的输入：

| M03 输出 | M04 消费 | 说明 |
|---|---|---|
| 日前预测曲线 | 收益模拟基线 | M04 基于预测出力计算策略收益 |
| 可调能力预测 | 策略可行性校验 | M04 生成策略时校验是否超出可调范围 |
| 精度指标 | 模拟置信度 | 预测精度影响收益模拟的置信区间 |

### 13.2 M03 → VPP 平台

可调能力上报链路：

```
M03 可调能力预测 → 聚合计算 → 上报接口 → VPP 平台接收
```

上报数据包含：
- 各电站未来 24h 可调能力时序
- 聚合系统级可调总量
- 可调置信度（基于预测精度）

### 13.3 M02 → M03 数据复用

M02 电站监控的"出力预测"Tab 使用 M03 的简化数据：
- M02 调用 `GET /api/pvms/stations/:id/forecast` 获取单站预测
- 数据来源与 M03 相同，但为单站视角、当日范围
- M02 该 Tab 底部提供"查看详细分析"按钮 → 跳转 M03

## 14. Mock 数据规划

Mock 数据文件：`src/modules/forecast/mock/forecast-mock.js`

### 14.1 Mock 数据策略

- 前端 Mock 先行，使用 `mockjs` 生成随机但合理的预测数据
- Mock 数据需符合物理约束：夜间出力为 0、白天出力呈钟形曲线、超短期预测误差小于日前预测
- 后续替换为真实后端 API

### 14.2 Mock 数据示例

```javascript
// forecast-mock.js
import Mock from 'mockjs'

// 生成 24h 预测曲线（15min 粒度，96 个点）
function generateForecastCurve() {
  const points = []
  for (let i = 0; i < 96; i++) {
    const hour = i / 4
    // 钟形曲线模拟光伏出力：6:00-18:00 有出力
    const base = hour >= 6 && hour <= 18
      ? Math.sin((hour - 6) / 12 * Math.PI) * 10
      : 0
    points.push({
      time: `${String(Math.floor(hour)).padStart(2, '0')}:${String((i % 4) * 15).padStart(2, '0')}`,
      dayAhead: +(base * (1 + (Math.random() - 0.5) * 0.2)).toFixed(2),
      ultraShort: +(base * (1 + (Math.random() - 0.5) * 0.1)).toFixed(2),
      actual: +(base * (1 + (Math.random() - 0.5) * 0.05)).toFixed(2)
    })
  }
  return points
}

// 生成偏差热力图数据
function generateHeatmapData(stations) {
  return stations.map(station => ({
    stationId: station.id,
    stationName: station.name,
    hourlyDeviation: Array.from({ length: 24 }, () =>
      +(Math.random() * 5 - 1).toFixed(2)
    )
  }))
}

// 生成可调能力时间线
function generateTimeline(stations) {
```
