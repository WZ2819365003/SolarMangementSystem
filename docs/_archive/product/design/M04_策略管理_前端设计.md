# M04 策略管理（前端设计）

> 模块定位：电站策略配置、电价接入、收益模拟与批量下发
> 用户角色：调度员、策略运营
> 技术栈：Vue 2 + Element UI + ECharts 5 + LESS
> 路由：`/strategy/list`、`/strategy/config`、`/strategy/revenue`
> 版本：V4.1（2026-03-27）
> 状态：✅ 前端已完成（全 Mock 数据，后端接口已定义未对接）

---

## 1. 设计结论

M04 作为策略管理模块，覆盖策略的完整生命周期：**配置 → 预览 → 下发 → 监控 → 收益统计**。本模块是前端系统中唯一的"写操作"密集型页面——前三个模块（M01/M02/M03）以"看"为主，M04 需要用户填写表单、配置参数、执行批量操作。

**核心职责**：
- 基于电站树结构，支持单站配置和批量配置策略
- 接入分时电价（自动获取 / 手动录入），驱动收益测算
- 调用 M03 预测数据验证策略可行性（是否超出可调范围）
- 模拟策略执行收益，支持多策略对比
- 管理策略生命周期：草稿 → 待执行 → 执行中 → 已完成

**核心业务链路**：

```
选择电站 → 录入响应信息 → 获取/配置电价 → 策略生成 → 收益模拟 → 确认下发
                                                          ↓
M03 预测数据 ──→ 可行性校验 ──→ 偏差警告（超出可调范围）
```

**跨模块关系**：
- M04 策略配置结果 → M02 电站监控「策略查看」Tab（只读展示）
- M03 预测数据 → M04 收益模拟基线 + 可行性校验
- M04 执行结果 → VPP 平台结算

**生产环境考量**：
- 所有策略操作通过标准 REST API 与后端交互，Mock 数据仅模拟响应格式
- 策略下发是关键操作，需二次确认弹窗 + 操作日志
- 电价数据接口预留自动获取通道，当前阶段以手动录入为主
- 批量操作需要后端事务支持，前端做好乐观锁 / 冲突提示

| # | 指标 | 单位 | accent | 说明 |

| 4 | 待处理策略 | 个 | orange | 草稿 + 待审批状态的策略数 |

### 4.2 筛选栏

| # | 控件 | 类型 | 说明 |
|---|---|---|---|
| 1 | 策略状态 | `el-select` | 全部 / 草稿 / 待执行 / 执行中 / 已完成 / 已终止 |
| 2 | 策略类型 | `el-select` | 全部 / 需求响应 / 调频辅助 / 电网约束 / 削峰填谷 |
| 3 | 电站 | `el-select` | 全部电站或指定电站 |

### 4.3 策略列表表格

| 列 | prop | 说明 |
|---|---|---|
| 勾选框 | selection | `el-table-column type="selection"` |
| 策略名称 | name | 策略名称，点击进入配置详情 |
| 所属电站 | stationName | 电站名称 |
| 策略类型 | type | 需求响应 / 调频辅助 / 电网约束 / 削峰填谷 |
| 状态 | status | el-tag 显示，不同颜色区分 |
| 执行时段 | timePeriod | 格式如 "08:00 - 18:00" |
| 目标功率(kW) | targetPowerKw | 策略目标功率 |
| 预估收益(元) | estimatedRevenue | 基于电价+预测的收益估算 |
| 创建时间 | createdAt | 策略创建时间 |
| 操作 | — | 查看 / 编辑 / 复制 / 终止 / 删除 |

### 4.4 策略状态定义

| 状态 | tag type | 说明 | 可执行操作 |
|---|---|---|---|
| 草稿 | info | 已保存未提交 | 编辑、删除、提交 |
| 待执行 | warning | 已提交待到执行时间 | 编辑、取消 |
| 执行中 | success | 正在执行 | 查看、终止 |
| 已完成 | — | 执行时段结束 | 查看、复制 |
| 已终止 | danger | 手动终止或异常终止 | 查看、复制 |

### 4.5 批量操作

表格支持多选（checkbox），底部显示批量操作栏：
- **批量下发**：将选中的草稿/待执行策略下发执行
- **批量暂停**：暂停选中的执行中策略
- **批量删除**：删除选中的草稿策略（仅限草稿状态）

批量操作需二次确认弹窗：
```
确认批量下发 3 个策略？

- 松山湖限功率策略（松山湖智造园A站）
- 武汉谷时填充策略（武汉物流园A站）
[取消]                              [确认下发]
```

## 5. 策略配置视图（Tab 2 — 核心）

路由：`/strategy/config`

回答的问题：如何为一个或多个电站配置响应策略？

### 5.1 左侧树面板

复用 M02 的 `StationTreePanel` 组件模式，但增加批量选择功能：

**单站模式**（默认）：
- 公司 → 电站 两级树（策略配置不需要逆变器层级）
- 点击电站节点，右侧面板加载该电站的策略配置
- 电站名旁显示策略状态图标：🟢 有执行中策略 / 🟡 有待执行策略 / ⚪ 无策略

**批量模式**（点击"批量选择"按钮切换）：
- 树节点前添加 `el-checkbox`
- 支持选择公司节点（自动选中全部下属电站）
- 底部显示 "应用到选中(N)" 按钮
- 批量配置时，策略参数统一填写，电站特定参数（如目标功率）可单独调整

### 5.2 步骤条

```
(1) 响应信息录入 → (2) 电价配置 → (3) 策略预览 → (4) 确认下发
```

使用 `el-steps` 组件，`process-status="finish"` 配合深色主题样式。

### 5.3 步骤 1：响应信息录入

表单字段：

| # | 字段 | 控件 | 校验 | 说明 |
|---|---|---|---|---|
| 1 | 策略名称 | `el-input` | 必填，2-30字 | 用户自定义名称 |
| 2 | 策略类型 | `el-select` | 必填 | 需求响应 / 调频辅助 / 电网约束 / 削峰填谷 |
| 3 | 执行日期 | `el-date-picker` | 必填，≥ 今天 | 策略执行的目标日期 |
| 4 | 执行时段 | 时段列表 | 至少1个时段 | 支持添加多个不重叠时段 |
| 5 | 目标功率 | `el-input-number` | 必填，>0 | 策略目标功率（kW） |
| 6 | 功率上限 | `el-input-number` | ≥ 目标功率 | 允许的最大功率（kW） |
| 7 | 功率下限 | `el-input-number` | ≤ 目标功率 | 允许的最小功率（kW） |
| 8 | 备注 | `el-input` textarea | 选填 | 策略说明 |

**时段配置**：
```
┌───────────────────────────────────────────────┐
│ 时段 1:  [08:00] 至 [12:00]              [删除] │
│ 时段 2:  [13:00] 至 [18:00]              [删除] │
│                              [+ 添加时段]       │
└───────────────────────────────────────────────┘
```
- 时间选择器精度到 15 分钟（与电价 96 点粒度对齐）
- 时段不可重叠，自动校验
- 最多 4 个时段

**可行性校验**（实时）：
- 调用 M03 可调能力数据，校验目标功率是否在可调范围内
- 显示校验结果：✅ 可行 / ⚠️ 接近限值 / ❌ 超出可调范围
- 超出可调范围时禁止进入下一步

## 9. 数据流

```
页面初始化:
  created() →
    fetchStrategyMeta()               // 获取策略类型、状态选项、电站列表
    loadTabData('list')               // 加载默认 Tab 数据

策略列表:
  loadStrategyList(filters) →
    fetchStrategyList(filters)        // 分页、筛选策略列表

策略配置（步骤式）:
  选择电站 →
    fetchStationStrategyDetail(stationId)     // 已有策略、可调范围
    fetchForecastAdjustable({ stationId })    // M03 可调能力（校验用）
  步骤1完成 → 本地缓存表单数据
  步骤2: fetchElectricityPrice({ date })      // 自动获取电价（或手动录入）
  步骤3: fetchStrategySimulation({ ... })     // 收益模拟
  步骤4: submitStrategy({ ... })              // 提交策略 → 后端创建

批量配置:
  选择多个电站 →
    统一填写策略参数（步骤1-2）
    fetchBatchSimulation({ stationIds, params })  // 批量收益模拟
    submitBatchStrategy({ strategies: [...] })     // 批量提交

收益分析:
  loadRevenueData(filters) →
    fetchStrategyRevenueSummary(filters)    // KPI + 趋势 + 饼图
    fetchStrategyRevenueDetail(filters)     // 明细表
  策略对比:
    fetchStrategyCompare({ ids: [id1, id2, id3] })  // 对比数据
```

## 15. 生产环境设计考量

M04 是"写操作"密集的模块，需要特别注意与后端的交互设计：

### 15.1 策略数据一致性

| 场景 | 前端处理 | 后端职责 |
|---|---|---|
| 策略创建 | 表单校验 + 可行性校验（调 M03） | 持久化 + 分配策略 ID + 状态机管理 |
| 策略编辑 | 乐观锁（携带 version 字段） | 版本比对 + 冲突拒绝 |
| 批量操作 | 前端显示进度（逐条反馈） | 事务处理 + 部分失败回滚 |
| 策略下发 | 二次确认 + loading 状态 | 异步执行 + WebSocket/轮询反馈 |

### 15.2 API 设计原则

- 所有写操作使用 POST/PUT/DELETE，不用 GET 修改数据
- 策略创建/更新的请求体包含完整数据（不依赖前端状态）
- 批量接口支持部分成功，响应包含每条策略的处理结果
- 收益模拟为无副作用的计算接口，可重复调用

### 15.3 Mock → 真实 API 迁移路径

```
当前阶段（Mock）：
  bridge.js → mockHandlers → 返回模拟数据

生产阶段（真实 API）：
  bridge.js → host API → 后端服务

  差异点：
  1. 策略 ID 由后端分配（Mock 用 UUID）
  2. 电价数据来自电力交易平台 API（Mock 用固定模板）
  3. 收益模拟由后端计算（Mock 用前端简化公式）
  4. 策略状态变更由后端状态机驱动（Mock 即时更新）
  5. 批量操作由后端事务保证（Mock 直接成功）
```

前端代码无需改动——只需在 `bridge.js` 中切换到真实 API 路径。
