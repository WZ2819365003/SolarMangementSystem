# 后端维护手册

## 1. 当前后端处于什么状态

当前后端不是“空 controller + 假数据”，也不是“已经生产化完成”的业务系统。

更准确的判断是：

- 它是一个可以直接运行的 Spring Boot 服务
- 关键模块已经进入 `H2 种子数据 + 后端计算` 模式
- 仍然存在少量开发态造数和待替换的数据源
- 当前重点不是继续堆前端 mock，而是稳住接口契约和数据边界

截至 `2026-03-30`，已经切到真实后端联调链路的重点模块有：

1. `dashboard`
2. `production-monitor`
3. `station-archive`
4. `forecast`
5. `strategy`

## 2. 启动入口和基础配置

启动类：

- `backend/src/main/java/cn/techstar/pvms/backend/PvmsBackendApplication.java`

配置文件：

- `backend/src/main/resources/application.yml`

当前关键配置：

- 端口：`8091`
- 应用名：`pvms-backend`
- 数据源：内嵌 `H2`

当前 H2 连接信息：

- JDBC URL：`jdbc:h2:mem:pvms;MODE=MySQL;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE`
- Username：`sa`
- Password：空

初始化脚本：

- `backend/src/main/resources/schema.sql`
- `backend/src/main/resources/data.sql`

## 3. 模块实现模式

### 3.1 H2 + 后端计算

这是当前最重要的模式，适用于：

- `dashboard`
- `production-monitor`
- `station-archive`
- `forecast`
- `strategy`

特点：

- H2 中保存基础事实数据或开发态元数据
- repository 负责查表
- service 负责装配、聚合和指标计算
- 前端开发态可以真实联调

### 3.2 开发态造数

当前仍然存在开发态种子数据和简化公式，但边界已经比过去清楚得多。

重点不是“完全没有造数”，而是：

- 哪些数据是元数据
- 哪些数据是系统事实数据
- 哪些结果必须由后端实时计算

## 5. H2 表分层怎么理解

### 5.1 事实表 / 主数据表

更接近系统事实的数据：

- `dashboard_*`
- `pm_*`
- `sa_*`

- `fc_station_model_binding`
- `fc_prediction_series_15m`

- `fc_adjustable_window`

### 5.3 策略元数据表

- `sg_strategy`
- `sg_strategy_period`
- `sg_execution_log`
- `sg_price_period`
- `sg_revenue_daily`
- `sg_strategy_snapshot`

## 6. 哪些结果必须由后端计算

这些结果不建议回退到前端拼装：

- dashboard 地图摘要和告警摘要
- production-monitor 的 KPI 和负荷汇总
- station-archive 的曲线聚合和 1 分钟插值
- forecast 的精度、偏差、可调上下界
- strategy 的收益模拟、收益汇总、树统计和对比

## 7. 当前最关键的限制

### 7.1 仍然大量使用 `Map<String, Object>`

优点：

- 开发快
- 联调灵活

缺点：

- 字段名没有编译期保护
- 契约漂移更依赖测试和联调发现

### 7.2 还没有完整权限和审计体系

当前不要默认已经具备：

- 登录
- 细粒度权限控制
- 操作审计闭环
- 调度任务治理

## 8. 新人第一次接手后端怎么做

1. 跑 `mvn test`
2. 打开 `http://127.0.0.1:8091/api/system/health`
3. 打开 `http://127.0.0.1:8091/h2-console`
4. 查看 `pm_* / sa_* / fc_* / sg_*` 表
5. 逐个手动访问关键接口

建议至少验证：

- `/api/pvms/production-monitor/load`
- `/api/pvms/station-tree`
- `/api/pvms/forecast/meta`
- `/api/pvms/forecast/overview?stationId=SZ-001`
- `/api/pvms/strategy/meta`
- `/api/pvms/strategy/list`
- `/api/pvms/strategy/simulate`

## 9. 后续建议

1. 给 `forecast` 和 `strategy` 补 DTO，逐步减少 `Map<String, Object>`
2. 把状态值、类型值和标签体系抽成统一枚举
3. 继续把收益、电价、执行日志替换成真实业务数据源
4. 在保持现有接口契约的前提下替换 repository 数据源
