# 后端维护手册

## 1. 当前后端处于什么状态

当前后端不是“纯接口壳”，也不是“已经生产化完成”的业务系统。

更准确的判断是：

- 它是一个可运行的 Spring Boot 服务
- 不同模块成熟度不同
- 一部分模块仍然是 mock / 内存态实现
- 一部分关键链路已经进入 “H2 种子数据 + 后端计算” 阶段

截至 `2026-03-30`，至少下面四条链路已经不再依赖前端 mock：

1. `dashboard` 地图动态点位链路
2. `production-monitor` 生产监控链路
3. `station-archive` 电站监控链路
4. `forecast` 预测分析链路

## 3. 模块实现模式

### 第一种：纯 mock / 演示装配

特征：

- controller 调 `*MockService`
- 数据不落库
- 主要用于页面演示或快速起页面

风险：

- 最容易和前端契约漂移
- 本地联调不容易暴露真实问题

### 第二种：H2 种子数据 + 后端计算

特征：

- 基础事实数据或元数据落在 H2
- repository 负责查表
- service 负责 KPI、图表、摘要、风险提示、分钟级拼装

当前已落地：

- `dashboard`
- `production-monitor`
- `station-archive`
- `forecast`

这是目前最值得继续演进的模式，因为它兼顾了：

- 本地真实联调
- 接口契约稳定
- 数据模型沉淀
- 后端承担计算职责
- 未来替换真实数据源的清晰边界

### 第三种：内存态业务服务

特征：

- 进程内维护状态
- 支持 create / submit / terminate 等动作
- 服务重启后状态丢失

当前典型模块：

- `strategy`

典型模块结构：

当前重点模块：

- `dashboard`

- `system`

## 5. 为什么要引入 H2

引入 H2 的目的不是做一个“临时数据库”，而是把下面这些能力从前端 mock 拿回后端：

- 接口契约校验
- 本地真实联调
- 指标计算
- 数据模型沉淀
- 未来替换真实系统时的边界保持

没有这层的话，页面虽然能展示，但无法验证：

- 查询参数是否真的参与计算
- 数据结构是否稳定
- 哪些字段是元数据
- 哪些字段是后端计算结果

## 7. H2 表分层怎么理解

### 事实表 / 主数据表

这一层更接近系统内真实业务事实：

- `dashboard_*`
- `pm_*`
- `sa_*`

### 预测元数据表

这一层用于预测模块开发态种子数据：

- `fc_model`
- `fc_station_model_binding`
- `fc_prediction_series_15m`
- `fc_error_sample`
- `fc_monthly_accuracy_snapshot`
- `fc_adjustable_window`

### 计算结果

不建议固化成表，应该继续由后端 service 计算：

- KPI 汇总
- 偏差热力图
- 精度趋势
- 可调能力上下界
- 风险提示
- 96 点 / 1 分钟图表序列

## 8. 当前最关键的限制

### 不是所有模块都已经数据化

不要因为 `dashboard`、`production-monitor`、`station-archive`、`forecast` 已经用了 H2，就误以为全仓都到同一成熟度。

目前依然是多种实现模式并存。

### 仍然大量使用 `Map<String, Object>`

优点：

- 改得快
- 联调灵活

缺点：

- 字段名没有编译期约束
- 契约漂移需要靠测试和联调发现

### 还没有完整权限与审计

当前不要默认认为已经具备：

- 登录
- 权限控制
- 操作审计
- 任务调度治理

## 9. 新人第一次接手后端建议怎么做

1. 跑 `mvn test`
2. 打开 `http://127.0.0.1:8091/api/system/health`
3. 打开 H2 Console
4. 查看 `dashboard_*`、`pm_*`、`sa_*`、`fc_*` 这些表
5. 手动请求下面几个接口

- `/api/pvms/dashboard/stations-geo`
- `/api/pvms/production-monitor/load`
- `/api/pvms/station-tree`
- `/api/pvms/station-archive/station-realtime?stationId=SZ-001&metric=adjustable`
- `/api/pvms/forecast/meta`
- `/api/pvms/forecast/overview?stationId=SZ-001`
- `/api/pvms/forecast/accuracy`
