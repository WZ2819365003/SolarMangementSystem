## 2. 启动入口和配置

### 启动类

`PvmsBackendApplication.java` — 使用 `@MapperScan("cn.techstar.pvms.backend")` 自动扫描所有 Mapper 接口。

### 配置文件

**生产环境** — `application.yml`

```yaml
server:
  port: 8091

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/pvms?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: "123456"

mybatis:
  mapper-locations: classpath:mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true
    arg-name-based-constructor-auto-mapping: true
```

**测试环境** — `application-test.yml`

```yaml
spring:
  datasource:
    driver-class-name: org.h2.Driver
    url: jdbc:h2:mem:pvms_test;MODE=MySQL;DB_CLOSE_DELAY=-1
    username: sa
    password:
  sql:
    init:
      mode: always
      schema-locations: classpath:schema.sql
      data-locations: classpath:data.sql
```

### 数据库初始化

```bash
# 建库
mysql -u root -p123456 -e "CREATE DATABASE IF NOT EXISTS pvms CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 建表
mysql -u root -p123456 --default-character-set=utf8mb4 pvms < src/main/resources/schema.sql

# 灌种子数据
mysql -u root -p123456 --default-character-set=utf8mb4 pvms < src/main/resources/mysql-data.sql
```

> `data.sql` 使用 H2 语法（`SYSTEM_RANGE` 等），仅供测试。MySQL 用 `mysql-data.sql`。

## 3. 架构分层

```
Controller
    ↓
Service（聚合、计算、业务逻辑）
    ↓
Mapper（@Mapper 接口 + XML）
    ↓
MySQL 8 / H2 (test)
```

### 关键设计决策

| 决策 | 说明 |
|---|---|
| Mapper 使用 Java `record` | 替代 POJO，不可变、简洁、自带 equals/hashCode |
| XML 使用 `resultType` 自动映射 | 配合 `map-underscore-to-camel-case`，无需手写 resultMap |
| 少量复杂映射用 `<constructor>` | 当列别名与字段名不匹配时使用 |
| Service 返回 `Map<String, Object>` | 灵活但无编译期保护，后续建议逐步替换为 DTO |
| `-parameters` 编译选项 | 保留方法参数名，MyBatis 可按名称匹配构造器参数 |

### 一个典型的 Mapper 长什么样

```java
@Mapper
public interface ProductionMonitorResourceUnitMapper {

    List<ResourceUnitRow> findAll();

    record ResourceUnitRow(
        String id, String name, String region, String city,
        String status, int clusterRadiusKm, ...
    ) {}
}
```

对应的 XML 文件在 `src/main/resources/mapper/productionmonitor/ProductionMonitorResourceUnitMapper.xml`：

```xml
<select id="findAll"
  resultType="...ProductionMonitorResourceUnitMapper$ResourceUnitRow">
    SELECT * FROM pm_resource_unit ORDER BY id
</select>
```

## 4. 数据库表分域

### 驾驶舱 (`dashboard_`)

| 表 | 说明 |
|---|---|
| `dashboard_station` | 电站基础信息（经纬度、容量） |
| `dashboard_station_status_snapshot` | 实时功率、发电量、健康等级 |
| `dashboard_alarm_snapshot` | 告警事件 |
| `dashboard_vpp_node_snapshot` | VPP 节点可调容量 |

### 生产监控 (`pm_`)

| 表 | 说明 |
|---|---|
| `pm_resource_unit` | 聚合单元（管理多个站点） |
| `pm_station` | 站点容量、权重、状态 |
| `pm_station_realtime_snapshot` | 实时快照 |
| `pm_weather_snapshot` | 气象条件 |
| `pm_station_curve_15m` | 15 分钟出力/负荷/预测曲线 |
| `pm_dispatch_record` | 调度指令记录 |

### 电站档案 (`sa_`)

| 表 | 说明 |
|---|---|
| `sa_company` | 公司/集团 |
| `sa_station` | 电站详情 |
| `sa_station_curve_15m` | 历史曲线 |
| `sa_inverter` | 逆变器台账 |
| `sa_inverter_alarm` | 逆变器告警 |
| `sa_station_strategy` | 站点执行策略 |

### 预测分析 (`fc_`)

| 表 | 说明 |
|---|---|
| `fc_model` | 预测模型定义 |
| `fc_station_model_binding` | 站点-模型绑定 |
| `fc_prediction_series_15m` | 预测序列 |
| `fc_error_sample` | 误差样本 |
| `fc_monthly_accuracy_snapshot` | 月度精度 |
| `fc_adjustable_window` | 可调窗口 |

### 策略管理 (`sg_`)

| 表 | 说明 |
|---|---|
| `sg_strategy` | 策略主表 |
| `sg_strategy_period` | 策略分时段 |
| `sg_execution_log` | 执行日志 |
| `sg_price_period` | 分时电价 |
| `sg_revenue_daily` | 日收益 |
| `sg_strategy_snapshot` | 模拟快照 |

## 5. 哪些结果必须由后端计算

这些结果不建议回退到前端拼装：

- Dashboard 地图摘要和告警汇总
- Production Monitor 的 KPI 和负荷聚合
- Station Archive 的曲线聚合和 1 分钟插值
- Forecast 的精度、偏差、可调上下界
- Strategy 的收益模拟、收益汇总和对比
- Device Alarm 的设备总览、分组比例和标签归一化

## 6. 目录速查

### 公共入口

```
backend/src/main/resources/
├── application.yml           # MySQL 生产配置
├── schema.sql                # DDL（兼容 MySQL 和 H2）
├── data.sql                  # H2 种子数据（测试用）
├── mysql-data.sql            # MySQL 种子数据（生产用）
└── mapper/                   # MyBatis XML 映射文件
    ├── dashboard/
    ├── devicealarm/
    ├── forecast/
    ├── productionmonitor/
    ├── stationarchive/
    └── strategy/
```

### 各模块入口

| 模块 | Controller | Service | Mapper |
|---|---|---|---|
| Dashboard | `dashboard/controller/DashboardController.java` | `dashboard/service/DashboardMapDataService.java` | `dashboard/repository/*Mapper.java` |
| Alarm | `dashboard/controller/AlarmController.java` | 同上 | 同上 |
| Weather | `dashboard/controller/WeatherController.java` | 同上 | 同上 |
| 生产监控 | `productionmonitor/controller/` | `productionmonitor/service/` | `productionmonitor/repository/` |
