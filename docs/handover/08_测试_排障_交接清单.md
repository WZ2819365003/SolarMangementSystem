# 测试、排障与交接清单

## 1. 当前已有的验证手段

### 后端

后端测试集中在：

- `backend/src/test/java/cn/techstar/pvms/backend/module/`

当前已覆盖模块：

- adjustable
- dashboard
- devicealarm
- forecast
- productionmonitor
- stationarchive
- stations
- strategy
- system

### 前端

前端当前主要有三种验证方式：

1. `npm run lint`
2. `npm run build`
3. Playwright smoke / layout 用例

Playwright 目录：

- `frontend/tests/playwright/`

当前可见用例包括：

- `dashboard-overview.spec.js`
- `forecast.spec.js`
- `layout-smoke.spec.js`
- `station-monitoring.spec.js`

## 2. 当前验证快照

以本次文档整理时的仓库状态为准：

- 后端 `mvn test` 可通过
- 前端 `npm run build` 可通过
- 前端 `npm run lint -- --no-fix` 未完全通过，仍有错误和较多 warning

这说明项目当前更像“能运行、能打包、后端测试可回归”，而不是“前后端联调完全稳定、前端质量门已关闭”。

## 3. 日常维护时建议执行的最小验证

### 只改前端样式或文案

```powershell
cd frontend
npm run build
```

### 改前端接口字段、页面逻辑或 mock

```powershell
cd frontend
npm run lint -- --no-fix
npm run build
```

必要时补充：

```powershell
cd frontend/tests/playwright
npm test
```

### 改后端接口或 mock service

```powershell
cd backend
mvn test
```

### 同时改前后端契约

```powershell
cd backend
mvn test

cd ../frontend
npm run lint -- --no-fix
npm run build
```

## 4. 常见问题排查

### 页面有数据，但后端接口明显不对

大概率原因：

- 前端当前用的是 `shared/mock`

先检查：

- `frontend/src/shared/host/bridge.js`

### 策略创建后数据重启消失

大概率原因：

- `strategy` 当前是内存态状态，不持久化

### 预测/策略页面联调时字段错位

大概率原因：

- 已知前后端契约漂移

### Playwright 起不来

优先检查：

- Node 是否为 18+
- 浏览器依赖是否安装
- `frontend` 是否已能正常 `serve`

### 安装依赖失败

优先检查：

- `frontend/package.json` 中的本地 tgz 依赖路径是否存在

## 5. 交接给下一位维护者前，至少要补齐的东西

### 文档层面

- 更新对应模块文档
- 如果改动了契约，必须更新交接文档和模块文档
- 如果新增了脚本或依赖，必须更新启动文档

### 验证层面

- 写清楚跑过哪些命令
- 如果没跑某项验证，要说明原因
- 如果当前仍有已知风险，要明确记录

### 代码层面

- 不要只改 mock 不改接口说明
- 不要只改前端不确认后端契约
- 不要在没有文档的情况下新增隐式约定

## 6. 新人接手的建议操作清单

### 第一步：确认环境

- Node 14 能起前端
- Java 21 + Maven 能起后端
- Node 18 能跑 Playwright

### 第二步：确认系统能运行

- 打开主页面
- 打开 forecast
- 打开 strategy
- 调用 health 接口

### 第三步：确认理解链路

- 找一个页面，从视图组件一路追到 `api/pvms.js`
- 再追到 `bridge.js`
- 再确认它当前是走前端 mock 还是后端接口

### 第四步：确认风险边界

- 读一遍 `07_Mock现状_联调策略_发布差距.md`
- 知道当前不能直接按生产系统思路判断代码状态

## 7. 推荐的发布前检查表

- 前端 lint 无阻断错误
- 前端 build 成功
- 后端测试通过
- 关键页面手工走查完成
- forecast / strategy 契约明确
- 发布环境配置明确
- 是否使用 mock 已明确记录
