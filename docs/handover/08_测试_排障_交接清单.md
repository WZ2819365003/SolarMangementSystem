## 4. 现在怎么排障

### 第一步：先判断是不是打到了真实后端

优先直接访问 `8091` 的接口。

如果 `8091` 正常但前端页面没数据，再看：

- 前端代理
- 页面状态
- 字段消费

### 第二步：确认 H2 是否真的有数据

打开：

- `http://127.0.0.1:8091/h2-console`

重点查看这些表组：

- `dashboard_*`
- `pm_*`
- `sa_*`
- `fc_*`

如果 `fc_*` 为空，优先检查：

- `schema.sql`
- `data.sql`
- `application.yml` 的 SQL 初始化配置

### 第三步：确认页面属于哪条后端链路

当前最容易混淆的是 M02 和 M03：

- 生产监控 -> `/api/pvms/production-monitor/*`
- 电站监控 -> `/api/pvms/station-tree`、`/api/pvms/station-archive/*`
- 预测分析 -> `/api/pvms/forecast/*`

如果接口前缀看错，排障方向会完全跑偏。

### 第四步：区分问题发生在元数据还是计算层

更像元数据问题的情况：

- 站点名称不对
- 区域筛选缺失
- 模型列表为空
- 电站数量不对

更像后端计算问题的情况：

- KPI 不合理
- 热力图数据全 0
- MAE / RMSE 不变化
- 可调上下界不对
- 比较曲线长度错误

## 5. Forecast 特别注意点

`forecast` 现在是分层取数：

- `sa_*` 提供事实数据
- `fc_*` 提供预测元数据
- `ForecastDataService` 负责聚合和计算

所以维护时不要只盯一个地方。

最少要同时看：

- `ForecastController`
- `ForecastDataService`
- `Forecast*Repository`
- `schema.sql`
- `data.sql`
- 前端 `ForecastPage.vue`

## 6. 交接前必须补齐什么

### 文档层面

- 更新对应模块文档
- 更新交接手册
- 如果改了表结构，要写清楚：
  - 哪些是基础事实表
  - 哪些是预测元数据表
  - 哪些指标是后端计算
  - 哪些数据仍然是开发态造数

### 验证层面

- 写清楚跑了哪些命令
- 写清楚验了哪些接口
- 如果没有跑前端或 Playwright，要明确说明

### 代码层面

- 不要只改前端，不确认后端契约
- 不要只补数据库表，不写 service 计算说明
- 不要把应由后端承担的聚合逻辑偷偷搬回前端

## 7. 对接手人的一句提醒

如果某个页面“看起来有数据”，先不要默认它已经是成熟链路。

先确认：

1. 前端是不是走了 `/api` 代理
2. 后端 controller 是不是真实现
3. 数据来自 H2 / 后端计算，还是仍停留在 mock
